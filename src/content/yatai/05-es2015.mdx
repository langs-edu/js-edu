この縁日で学ぶJavaScriptは「モダンJavaScript」と呼ばれる書き方です。では、モダンはいつ始まったのか。答えははっきりしていて、2015年6月です。

この月、ECMAScript(JavaScriptの標準仕様)の第6版が承認されました。正式名はECMAScript 2015、通称ES2015、古い呼び名でES6。この一冊に、いまのJavaScriptの顔ぶれがほとんど揃っていました。`let`と`const`、テンプレートリテラル、アロー関数、`class`、分割代入、`for...of`、モジュール——このサイトで学ぶ道具の多くが、この年に一度にやってきたのです。

「一度に」というのがポイントです。ひとつ前の大きな改訂は1999年の第3版で、その次の第5版は2009年。言語をどう進化させるべきかをめぐって標準化委員会が長く割れ、10年近く新機能が止まった時期がありました。ES2015は、その停滞の間に溜まりに溜まった宿題を一気に片づけた大改訂だったのです。

たとえば文字列の組み立ては、こう変わりました。

```js
// ES2015より前の書き方(いまも動きます)
var name = "たろう";
var greeting = "こんばんは、" + name + "さん";
console.log(greeting);
```

```js
// ES2015からの書き方
const name = "たろう";
const greeting = `こんばんは、${name}さん`;
console.log(greeting);
```

そしてもうひとつの転換が、リリースの仕方そのものでした。何年分もの変更をためてから大改訂するのはやめて、毎年6月に小さく改訂する。ES2016以降、仕様は年号を名前にして毎年出ています。10年に一度の大祭をやめて、毎年の縁日にした、と言えるかもしれません。

だから安心してください。この縁日で覚える書き方は10年来の標準であり、これからも毎年の改訂を受け止める土台であり続けるものです。
