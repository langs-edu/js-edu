「JavaScriptって、Javaの仲間ですか?」——この質問は、30年間ずっと繰り返されてきました。答えははっきりしています。ふたつは別の言語です。作った人も、設計の考え方も違います。

では、なぜこんなに紛らわしい名前なのでしょうか。

1995年、Netscape社の新しい言語は、LiveScriptという名前でブラウザのベータ版に載っていました。ところが同年12月4日、NetscapeはSun Microsystems社との提携を発表し、あわせて言語名をJavaScriptに改めます。Sunが開発したJavaは、当時のIT業界で一番の話題をさらっていた言語でした。その勢いにあやかった、マーケティング上の命名だったと言われています。JavaScriptという商標はSunが所有し、Netscapeはそれを借りる形をとりました。

中身はどうかというと、構文の見た目こそJavaに寄せられたものの、言語の心臓部はまったくの別物です。JavaScriptでは関数を値として扱う発想はSchemeから、クラスを使わないオブジェクトの仕組みはSelfから受け継がれました。開発の現場ではよく「メロンとメロンパンくらい違う」とたとえられますが、実際そのくらいの間柄です。

名前の物語には続きがあります。商標は、Sunを買収したOracle社に2010年に引き継がれました。そして2024年、「JavaScriptはもはや誰のものでもない一般名称なのだから、商標を手放すべきだ」とする申し立てが米国で起こされます。賛同者には、JavaScriptの作者Brendan Eich本人も名を連ねました。この原稿を書いている2026年7月時点で、決着はまだついていません。生まれて30年経っても、この言語は自分の名前をめぐって揉めているのです。

ちなみに、名前を貸した側のJavaの物語は、姉妹サイトの[喫茶Java](https://langs-edu.github.io/java-edu/)で読めます。名前だけの他人とはいえ、あちらもよい言語ですよ。
