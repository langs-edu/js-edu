## 名前がついた値には、種類がある

前回、`const`や`let`で値に名前をつけました。`"りんご飴"`と`3`は、見るからに違う種類の値です。この「種類」のことを**型**と呼びます。JavaScriptには、`typeof`という演算子があり、値の型をその場で確かめられます。

<CodeRunner client:visible storageKey="hajimete-04-a" initialCode={`const kosuu = 3;
const na = "りんご飴";
const uritekiru = true;
console.log(typeof kosuu);
console.log(typeof na);
console.log(typeof uritekiru);`} />

出力欄には、`number`・`string`・`boolean`という3つの言葉が並びます。それぞれ、JavaScriptの基本的な型の名前です。

## number——整数も小数も、ひとつの型

`number`は数を表す型です。ここで、他の言語を少しでも触ったことがある人にとっては意外に感じる点があります。**JavaScriptには、整数専用の型と小数専用の型の区別がありません。** `3`も`3.14`も、どちらも同じ`number`型です。

<CodeRunner client:visible storageKey="hajimete-04-b" initialCode={`console.log(typeof 3);
console.log(typeof 3.14);`} />

両方とも`number`と表示されます。整数か小数かは、値そのものの見た目の違いであって、型としては区別されません。

## string——文字列

`string`は、`"りんご飴"`のように、ダブルクォート（またはシングルクォート）で囲んだ文字の並びです。数がいくつであろうと、クォートで囲めばそれは`string`になります。

<CodeRunner client:visible storageKey="hajimete-04-c" initialCode={`console.log(typeof "3");
console.log(typeof 3);`} />

`"3"`と`3`は、見た目こそ似ていますが、型はまったく違います。前者は`string`、後者は`number`です。この違いは、この先の計算のところで実感を持って効いてきます。

## boolean——真か偽か

`boolean`は、`true`（真）か`false`（偽）のどちらかしか持てない型です。「営業中かどうか」のような、2択の状態を表すのに使います。`true`・`false`はどちらも小文字で書きます。

## 型は、値のほうについている

一度つけた変数名に、別の型の値を代入し直すとどうなるでしょうか。`let`で試してみます。

<CodeRunner client:visible storageKey="hajimete-04-d" initialCode={`let x = 10;
console.log(typeof x);
x = "じゅう";
console.log(typeof x);`} />

エラーは出ません。`x`の型が`number`から`string`へ、静かに変わっただけです。JavaScriptでは、変数を宣言するときに型を書きません。**型は変数の名前についているのではなく、そのとき代入されている値のほうについています。** `x`という名前が、そのときどきで違う型の値を指し示せる——これがJavaScriptの動的型付けという性質です。

<Callout type="note" title="「型がない」わけではない">
これは「JavaScriptに型がない」という意味ではありません。値そのものには、確かに`number`や`string`という型があります。ただ、その型を変数の宣言時に固定しないだけです。`typeof x`で聞いているのは、いつも「今`x`が指している値の型」であって、`x`という名前そのものに刻まれた型ではありません。
</Callout>

## 昔からある書き方——var

ここで、`const`や`let`より古くから存在する`var`という書き方にも、一度だけ触れておきます。

```js
var mukashi = "古い書き方もできます";
console.log(mukashi);
```

`var`も変数を宣言でき、今でも動きます。古いコードの中では今も見かけるので、読めるようにはしておきたい書き方です。ただし、`const`や`let`と比べて挙動にクセがあり、意図しない不具合の原因になりやすいため、**このサイトでは自分から書くことはしません。** 基本は`const`、付け替えたいときは`let`、という前回の方針のままで進めます。

## 0.1 + 0.2 という夜店の手品

数の計算では、ときどき不思議なことが起こります。試してみましょう。

<CodeRunner client:visible storageKey="hajimete-04-e" initialCode={`console.log(0.1 + 0.2);
console.log(0.1 + 0.2 === 0.3);`} />

出力は、次のようになります。

```console
0.30000000000000004
false
```

`0.1 + 0.2`は、`0.3`ぴったりにはならず、ほんのわずかにずれた値になります。これはJavaScriptに限った不具合ではなく、コンピュータが小数を扱う仕組みそのものに由来する、多くの言語に共通する性質です。仕組みの詳しい説明はここでは深入りしませんが、「小数の計算には、ごくわずかな誤差がつきものだ」ということだけ、誠実に知っておいてください。🏮 種明かしは屋台のこぼれ話「[0.1 + 0.2 の夜店の手品](../../../yatai/08-floating-point/)」でどうぞ。

<Callout type="warn" title="つまずきポイント：小数どうしをぴったり比較しない">
`0.1 + 0.2 === 0.3`が`false`になるように、小数の計算結果を`===`でぴったり比較すると、期待どおりにならないことがあります。実務では、差がごく小さい範囲に収まっているかどうかで比較する、といった工夫が必要になりますが、このコースの範囲を超えるので、今は「そういう性質がある」と知っておくだけで十分です。
</Callout>

## このレッスンで分かったこと

- `typeof`で、値の型をその場で確かめられる
- `number`は整数も小数もひとつの型。`string`はクォートで囲んだ文字列。`boolean`は`true`/`false`のどちらか
- 型は変数の名前ではなく、値のほうについている。同じ変数名に、違う型の値を自由に代入し直せる
- `var`は古いコードで見かける書き方。読めればよく、自分では書かない
- 小数の計算には、`0.1 + 0.2 === 0.3`が`false`になるような、ごくわずかな誤差がつきものである
