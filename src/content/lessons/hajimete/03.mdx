## 値に、名前をつける

前回まで、`console.log("りんご飴")`のように、値をその場で直接書いてきました。でも、同じ値を何度も使いたくなったら、そのたびに書き直すのは面倒です。値に**名前**をつけて取っておければ、あとから何度でもその名前で呼び出せます。

<CodeRunner client:visible storageKey="hajimete-03-a" initialCode={`const yatai = "りんご飴";
console.log(yatai);`} />

`const yatai = "りんご飴";`という1行が、`"りんご飴"`という値に`yatai`という名前をつけています。これを**変数の宣言**と呼びます。次の行では、`yatai`という名前を書くだけで、さっき取っておいた値を呼び出せています。

**変数とは、値につけた名前です。** 「値を箱に入れる」という説明を見かけることがありますが、この縁日ではその言い方をしません。この先、1つの値に2つ以上の名前がつく場面も出てきます。名前は、あだ名のように、いくつあっても不思議ではありません。

## constが基本、letは付け替えたいときだけ

`const`は「constant（定数）」の略で、**一度つけた名前には、二度と別の値を代入し直さない**という宣言です。このサイトでは、`const`を基本の書き方とします。値が変わらないと分かっているなら、そう宣言しておいたほうが、あとでコードを読む人（未来のあなた自身も含みます）にとって親切だからです。

一方で、名前を使い回して値を付け替えたい場面もあります。そのときは`let`を使います。

<CodeRunner client:visible storageKey="hajimete-03-b" initialCode={`let kingyo = 3;
console.log(kingyo);
kingyo = 5;
console.log(kingyo);`} />

`let kingyo = 3;`で名前をつけたあと、`kingyo = 5;`のように、`let`をつけずに名前だけ書いて別の値を代入し直しています。これを**再代入**と呼びます。出力欄には、`3`のあとに`5`が表示されます。同じ`kingyo`という名前が、途中から違う値を指すようになったわけです。

## constに再代入すると、どうなるか

`const`で宣言した名前に、同じように再代入しようとすると何が起きるでしょうか。試してみましょう。このコードは、実行するとエラーで止まる例です。

<CodeRunner client:visible storageKey="hajimete-03-c" expectError initialCode={`const kingyo = 3;
kingyo = 5;`} />

実行すると、次のようなメッセージが表示されます。

```console
TypeError: Assignment to constant variable.
```

「定数変数への代入です」——`const`で宣言した名前には、二度と代入できないという約束を破ろうとしたことを、JavaScriptがその場で教えてくれています。これは、このサイトで初めて出会う本物のエラーです。エラーが出ること自体は失敗ではありません。**「今、何をしようとして、なぜ止まったのか」を読み取れれば、それで十分です。** エラーの種類ごとの詳しい読み方は、コースの後半（レッスン7）でまとめて扱います。

<Callout type="note" title="迷ったらconst">
「これから値を変えるかもしれない」と少しでも迷ったときは、いったん`const`で書いてみてください。実際にコードを書き進めていく中で、「やっぱり再代入したい」と分かった時点で`let`に書き換えれば十分です。最初から`let`を使いすぎると、どの変数が本当に変わる予定なのか、コードを読むだけでは分かりにくくなります。
</Callout>

## 名前のつけ方——camelCase

変数名は、日本語ではなくローマ字か英単語で、しかも自由な文字列というわけではありません。数字から始めることはできず、スペースも使えません。複数の単語からなる名前をつけるときは、2つ目以降の単語の先頭を大文字にしてつなげます。これを**camelCase**（キャメルケース）と呼びます。

<CodeRunner client:visible storageKey="hajimete-03-d" initialCode={`const uriageKingaku = 500;
console.log(uriageKingaku);`} />

`uriageKingaku`のように、単語の境目が大文字になっている見た目が、こぶが並んだラクダ（camel）に似ていることからこの名前がついています。JavaScriptの変数名は、このcamelCaseで書くのが主流です。

## このレッスンで分かったこと

- 変数は、**値につけた名前**。`const 名前 = 値;`や`let 名前 = 値;`で宣言する
- 基本は`const`。値を付け替えたいときだけ`let`を使う
- `const`で宣言した名前に再代入しようとすると、`TypeError: Assignment to constant variable.`というエラーになる
- 変数名は数字から始められず、複数単語はcamelCase（`uriageKingaku`のように）でつなげる
