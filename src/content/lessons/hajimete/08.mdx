## ブラウザの外へ

コース1の最初のレッスンで、「JavaScriptはブラウザの中だけの言語ではなくなった」と少しだけ触れました。ここで、その舞台を実際に覗いてみます。**Node.js**は、ブラウザを開かなくても、パソコンやサーバーの上で直接JavaScriptを実行できる仕組みです。Webページの中で動くための言語だったJavaScriptが、Node.jsのおかげで、ファイルの読み書きや、サーバーを立てるようなプログラムにも使われるようになりました。

このレッスンだけは、CodeRunnerの中ではなく、あなたのパソコンで実際に手を動かします。環境が用意できなくても、読むだけで流れはつかめるように書いてあります。

## Node.jsをインストールする

公式サイト（[nodejs.org](https://nodejs.org/)）を開くと、ダウンロードボタンがいくつか並んでいます。中でも**LTS**（Long-Term Support、長期サポート版）と書かれたボタンを選んでください。WindowsでもmacOSでも、ダウンロードしたインストーラを実行し、案内される画面のとおりに進めれば完了します。Linuxの場合は、ディストリビューションのパッケージ管理ツールを使う方法もありますが、迷ったら公式サイトの案内に従うのが確実です。

<Callout type="note" title="バージョンについて">
このサイトのコードはNode.js 22で動作確認していますが、それより新しいLTS版（インストール時にもっと大きな番号が案内されることもあります）でも、このサイトのコードは問題なく動きます。迷ったら、公式サイトで**LTS**と書かれているほうを選んでください。
</Callout>

## インストールできたか確かめる

ターミナル（WindowsはコマンドプロンプトやPowerShell）を開いて、次のコマンドを実行してください。

```console
node --version
```

インストールできていれば、`v22.14.0`のような形式で、バージョン番号が表示されます。数字の部分は、あなたがインストールした時期によって変わります。先頭が`v`で始まる番号らしきものが表示されていれば成功です。もし`command not found`のようなメッセージが出た場合は、インストールがうまくいっていないか、一度ターミナルを閉じて開き直す必要があるかもしれません。

## hello.jsを書いて、実行する

コードをファイルに保存してみましょう。エディタで新しいファイルを作り、`hello.js`という名前で、次の1行を保存してください。

```js title="hello.js"
console.log("いらっしゃい、縁日へ");
```

保存したら、ファイルを保存したフォルダでターミナルを開き、次のコマンドを実行してください。

```console
$ node hello.js
いらっしゃい、縁日へ
```

`node ファイル名`という形で、そのファイルに書かれたJavaScriptが実行されます。ここまでのレッスンでCodeRunnerに書いてきたコードと、書き方はまったく同じです。実行する場所が、ブラウザの中からターミナルに変わっただけです。

## ブラウザと共通の部分、違う部分

JavaScriptという言語そのもの、`console.log`のような基本的な仕組みは、ブラウザでもNode.jsでも変わりません。ここまでの7つのレッスンで学んだことは、すべてそのままNode.jsでも通用します。

一方で、違う部分もあります。ブラウザだけが持っている`document`のような仕組みは、Node.jsには存在しません。試しに、次のコードをファイルに保存して実行してみてください。

```js expect-error title="mado.js"
console.log(document);
```

実行すると、次のようなエラーになります（実際の画面には、この上下にファイルの場所や詳しい呼び出し履歴も表示されますが、ここでは要点だけを示しています。上の行は、あなたが保存した場所によって変わります）。

```console
$ node mado.js
ReferenceError: document is not defined
```

`document`は、Webページというものが存在するブラウザの中でだけ意味を持つ仕組みです。ページを表示していないNode.jsには存在しません。逆に、Node.jsには`fs`（ファイルを読み書きする道具）や`require`（他のファイルのコードを読み込む仕組み）のような、ブラウザには無い道具が用意されています。これらの中身は、この先のコース6でモジュールを学ぶときにあらためて扱います。今は名前だけ知っておいてください。

<Callout type="warn" title="つまずきポイント：ReferenceErrorは、前回学んだのと同じ顔">
`document is not defined`というメッセージは、レッスン7で見た`mikan is not defined`とまったく同じ形をしています。「存在しない名前を使おうとした」という意味では、原因はまったく同じです。`document`という名前が「存在しない」のは、間違えたからではなく、**今いる場所（Node.js）にはその仕組みがそもそも用意されていないから**です。同じReferenceErrorでも、原因はタイプミスとは限らない、ということも知っておいてください。
</Callout>

## このレッスンで分かったこと

- **Node.js**は、ブラウザの外でJavaScriptを実行できる仕組み。公式サイトからLTS版をインストールする
- `node ファイル名`で、ファイルに保存したJavaScriptを実行できる
- 言語そのものや`console.log`はブラウザと共通。`document`のようなブラウザ専用の仕組みはNode.jsには無く、代わりに`fs`や`require`のような道具がある
- 以後のコースは、基本的にこのサイトのCodeRunnerで進める
