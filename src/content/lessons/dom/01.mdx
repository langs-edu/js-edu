## これまでの相手は、コンソールだけでした

ここまでの4つのコースで、値の種類、条件分岐、くりかえし、配列とオブジェクト、関数とclassまで、JavaScriptという言語そのものを見てきました。書いたコードの行き先は、いつも`console.log`でした。

このコースからは、行き先が変わります。JavaScriptが本来生まれた場所——**Webページ**です。ボタンを押すと文字が変わる、入力欄に書いた名前が挨拶に反映される、そんな「動くページ」の裏側を、これから6レッスンで作れるようになります。まずは、その舞台であるHTMLの読み方からです。

## HTMLは、入れ子でできている

HTMLは、**タグ**で文章の意味を区切っていく言語です。開始タグと終了タグで挟まれた部分が、1つの**要素**になります。

```html
<p>今夜は縁日です。</p>
```

`<p>`が開始タグ、`</p>`が終了タグ、その間の「今夜は縁日です。」が中身です。要素は入れ子にできます。

```html
<div>
  <h1>縁日へ、ようこそ</h1>
  <p>今夜は屋台がいっぱいです。</p>
</div>
```

`div`という要素の中に、`h1`と`p`という要素が入っています。この「外側と内側」の関係が、このあとの話の主役になります。

要素には、**属性**という追加情報をつけられます。このコースでよく使うのは`id`と`class`の2つです。

```html
<p id="intro" class="message">今夜は屋台がいっぱいです。</p>
```

- `id`——その要素だけにつける名札。**1つの文書の中で、同じidは1つの要素にしか使いません**
- `class`——グループにつける名札。**同じclassを、複数の要素につけてかまいません**

この2つは、次のレッスンで「欲しい要素を取り出す」ための手がかりになります。

<Callout type="mame" title="タグは全部終了タグが要るわけではない">
`<p>`や`<div>`は開始タグと終了タグの対ですが、`<img>`や`<input>`のように、中身を持たず終了タグを書かないタグもあります。`input`はレッスン05で登場します。
</Callout>

## ブラウザは、HTMLを「木」として持っている

ブラウザは、受け取ったHTMLの文字列をそのまま画面に貼り付けているわけではありません。タグの入れ子関係を読み取って、要素どうしの親子関係でできた**木構造**をメモリの中に作ります。これを**DOM**（Document Object Model、文書オブジェクトモデル）と呼びます。

さきほどの縁日のHTMLを、少し増やしてみます。

```html
<h1>縁日へ、ようこそ</h1>
<p id="intro">今夜は屋台がいっぱいです。</p>
<ul>
  <li class="yatai">りんご飴</li>
  <li class="yatai">金魚すくい</li>
  <li class="yatai">ラムネ</li>
</ul>
<button id="entry-btn">入り口はこちら</button>
```

これをブラウザが木にすると、次のような形になります。

```text
document
└─ html
   └─ body
      ├─ h1 「縁日へ、ようこそ」
      ├─ p#intro 「今夜は屋台がいっぱいです。」
      ├─ ul
      │  ├─ li.yatai 「りんご飴」
      │  ├─ li.yatai 「金魚すくい」
      │  └─ li.yatai 「ラムネ」
      └─ button#entry-btn 「入り口はこちら」
```

`html`や`body`は、上のHTMLには書いていません。ブラウザは、書き忘れられていても文書全体を`html`と`body`で自動的に囲みます。この木の一番上、すべての要素への入り口になっているオブジェクトが**document**です。次のレッスンから、`document`を使ってこの木の中から欲しい枝を取り出していきます。

<Callout type="note" title="documentはJavaScriptの言葉ではない">
`document`はJavaScriptという言語そのものが持っている機能ではありません。**ブラウザが**、実行中のJavaScriptに向けて用意している入口です。コース1の最後で「Node.jsにはdocumentが無い」と学んだのを覚えているでしょうか。ブラウザという舞台があってはじめて、`document`は意味を持ちます。
</Callout>

## このレッスンで分かったこと

- HTMLは、開始タグと終了タグで挟まれた**要素**が入れ子になってできている
- `id`は1つの要素だけにつける名札、`class`は複数の要素につけてよいグループの名札
- ブラウザはHTMLを、要素どうしの親子関係でできた**木構造**（DOM）としてメモリに持つ
- `document`は、その木の一番上にある入口。JavaScriptから木の中に触れるための出発点
- このコースからは**HtmlRunner**で、HTMLとJavaScriptをまとめて動かしながら学ぶ
