## ここまでの道具を、1つにまとめる

このコースで見てきた道具を、並べておきます。

- `document.querySelector`で、木の中から要素を取る
- `textContent`で、中身を読み書きする
- `classList`で、見た目のクラスを付け外しする
- `addEventListener`で、クリックに応えて動く
- `input.value`で、読み手の入力を受け取る

この5つを組み合わせれば、動く小さな道具が作れます。このレッスンでは、**クリックカウンター**と**おみくじ**という2つの遊び道具を作りながら、コース5の総仕上げをします。

## クリックカウンター

押すたびに数字が増えるボタンです。

<HtmlRunner
  client:visible
  storageKey="dom-06-a"
  height={160}
  initialHtml={`<button id="counter-btn">押す</button>
<p id="counter-display">0回</p>
<script>
  let count = 0;
  const button = document.querySelector("#counter-btn");
  const display = document.querySelector("#counter-display");
  button.addEventListener("click", () => {
    count = count + 1;
    display.textContent = count + "回";
  });
</script>`}
/>

`count`という変数は、クリックハンドラの**外**で1回だけ`let count = 0;`と宣言されています。ボタンが押されるたびに呼ばれる関数は、その外側にある`count`を`count = count + 1;`で書き換え続けます。コース2のスコープの回で見た「外の変数への代入」が、ここで実際に働いています。もし`count`をハンドラの中で`let count = 0;`と宣言し直していたら、押すたびに0からやり直しになってしまい、カウンターにはなりません。

<Callout type="warn" title="つまずきポイント：変数の宣言位置">
数え続けたい値は、イベントハンドラの**外側**で1回だけ宣言してください。ハンドラの中で`let`や`const`を書くと、クリックのたびに新しく作り直されてしまい、前の値を覚えていられません。
</Callout>

## おみくじ——コース2・3の道具を、DOMで使う

おみくじは、配列からランダムに1つ選ぶだけの仕組みです。まずはDOM抜きで、ロジックだけ確認しておきます。

<CodeRunner
  client:visible
  storageKey="dom-06-b"
  initialCode={`const fortunes = ["大吉", "吉", "中吉", "小吉", "凶"];
const index = Math.floor(Math.random() * fortunes.length);
console.log(fortunes[index]);`}
/>

`Math.random()`は0以上1未満の小数を返すので、それに`fortunes.length`（5）を掛けて`Math.floor`で整数にすると、0から4までの添字がランダムに手に入ります。コース2の数当てゲーム、コース3の配列——その組み合わせです。実行するたびに表示される運勢は変わりますが、どの実行でもエラーなく終わります。

これをDOMにつなげると、こうなります。

<HtmlRunner
  client:visible
  storageKey="dom-06-c"
  height={160}
  initialHtml={`<button id="omikuji-btn">おみくじを引く</button>
<p id="omikuji-result">ボタンを押してください</p>
<script>
  const fortunes = ["大吉", "吉", "中吉", "小吉", "凶"];
  const button = document.querySelector("#omikuji-btn");
  const result = document.querySelector("#omikuji-result");
  button.addEventListener("click", () => {
    const index = Math.floor(Math.random() * fortunes.length);
    result.textContent = fortunes[index];
  });
</script>`}
/>

押すたびに、5つの運勢のどれかがランダムに表示されます。ロジックの部分は、さきほどのCodeRunnerとまったく同じです。違うのは、結果を`console.log`ではなく`result.textContent`に渡している点だけです。

## 2つを、1つのページに

このコースの最後に、2つを1つのページにまとめてみます。

<HtmlRunner
  client:visible
  storageKey="dom-06-d"
  height={260}
  initialHtml={`<h2>縁日の遊び場</h2>
<button id="counter-btn">押す</button>
<p id="counter-display">0回</p>
<hr>
<button id="omikuji-btn">おみくじを引く</button>
<p id="omikuji-result">ボタンを押してください</p>
<script>
  let count = 0;
  const counterBtn = document.querySelector("#counter-btn");
  const counterDisplay = document.querySelector("#counter-display");
  counterBtn.addEventListener("click", () => {
    count = count + 1;
    counterDisplay.textContent = count + "回";
  });

  const fortunes = ["大吉", "吉", "中吉", "小吉", "凶"];
  const omikujiBtn = document.querySelector("#omikuji-btn");
  const omikujiResult = document.querySelector("#omikuji-result");
  omikujiBtn.addEventListener("click", () => {
    const index = Math.floor(Math.random() * fortunes.length);
    omikujiResult.textContent = fortunes[index];
  });
</script>`}
/>

1つのページの中に、2組の`querySelector`と`addEventListener`が、それぞれ独立して動いています。カウンターのボタンを押しても、おみくじの結果は変わりません。それぞれの要素とイベントが、別々に結び付いているからです。

## コース5で分かったこと

- `document.querySelector`で木から要素を取り、`textContent`で中身を読み書きする
- `classList.add/remove/toggle`で見た目のクラスを付け外しする
- `addEventListener("click", 関数)`で、クリックという「あとで起きること」に応える。渡す関数はコース4の「値としての関数」そのもの
- `input.value`は常に文字列。`Number()`で変換してから計算する
- イベントハンドラをまたいで値を覚えておきたいときは、ハンドラの**外側**で変数を宣言する
