## もう一度、textContentで書き換える

前回、`querySelector`で取った要素の`textContent`に代入すると、中身がまるごと置き換わることを見ました。これを使えば、最初に書いておいたHTMLの文字を、JavaScriptが書き換えることができます。

<HtmlRunner
  client:visible
  storageKey="dom-03-a"
  height={140}
  initialHtml={`<p id="status">準備中...</p>
<script>
  const status = document.querySelector("#status");
  status.textContent = "本日20時開店！";
</script>`}
/>

ページが読み込まれると、「準備中...」だった文字は「本日20時開店！」に置き換わります。今はページが読み込まれた瞬間に書き換えていますが、次のレッスンでは、これを「ボタンを押した瞬間」に書き換えるやり方に進みます。

## 見た目を切り替える——classList

文字だけでなく、見た目そのものを切り替えたいこともあります。そのための道具が`classList`です。要素の`class`属性を、あとからJavaScriptで付けたり外したりできます。

<Callout type="mame" title="このコースでは、CSSの書き方そのものは教えません">
`<style>`の中に書く`.highlight { background: #fff3b0; }`のようなルールは、色や背景を決めるCSSという別の言語です。このコースでは「あるクラス名がついていると、こういう見た目になる」という結びつきだけ使い、CSSの文法そのものは扱いません。
</Callout>

`classList`には主に3つのメソッドがあります。

| メソッド | 動作 |
|---|---|
| `classList.add("名前")` | そのクラスを付ける（すでに付いていれば何もしない） |
| `classList.remove("名前")` | そのクラスを外す（付いていなければ何もしない） |
| `classList.toggle("名前")` | 付いていれば外し、付いていなければ付ける |

まず`remove`です。

<HtmlRunner
  client:visible
  storageKey="dom-03-b"
  height={140}
  initialHtml={`<style>
  .highlight { background: #fff3b0; }
</style>
<p id="yatai" class="highlight">りんご飴 300円</p>
<script>
  const yatai = document.querySelector("#yatai");
  yatai.classList.remove("highlight");
</script>`}
/>

この段落には、最初`class="highlight"`が付いていて、黄色い背景になるはずでした。しかしJavaScriptが`classList.remove("highlight")`を実行するので、最終的には`highlight`クラスが外れ、背景の色は付きません。

`add`はその逆です。

<HtmlRunner
  client:visible
  storageKey="dom-03-c"
  height={140}
  initialHtml={`<style>
  .highlight { background: #fff3b0; }
</style>
<p id="yatai">かき氷 350円</p>
<script>
  const yatai = document.querySelector("#yatai");
  yatai.classList.add("highlight");
</script>`}
/>

今度は最初から`highlight`クラスは付いていませんが、`classList.add("highlight")`が実行されるので、最終的に黄色い背景が付きます。

`toggle`は、今その要素にクラスが付いているかどうかを見て、自動的に逆を選びます。

<HtmlRunner
  client:visible
  storageKey="dom-03-d"
  height={140}
  initialHtml={`<style>
  .highlight { background: #fff3b0; }
</style>
<p id="yatai" class="highlight">ラムネ 200円</p>
<script>
  const yatai = document.querySelector("#yatai");
  yatai.classList.toggle("highlight");
</script>`}
/>

この段落は最初から`highlight`が付いているので、`toggle`は「付いているなら外す」を選び、最終的に黄色い背景は消えます。もし最初から`highlight`が付いていなければ、`toggle`は逆に「付ける」を選んでいたはずです。**今の状態を自分で確かめなくても、trueかfalseで自動的に切り替えてくれる**のが`toggle`の便利さです。次のレッスンでは、これをボタンのクリックと組み合わせて、押すたびに見た目が切り替わる様子を作ります。

<Callout type="note" title="toggleには、強制的に決める書き方もある">
`classList.toggle("highlight", true)`のように第2引数を渡すと、今の状態に関係なく「trueなら必ず付ける、falseなら必ず外す」という動きになります。「ある条件を満たしていたら、このクラスを付けたい」というときに便利です。
</Callout>

## style直接操作というもう1つの道具

`classList`のほかに、`el.style.color = "red";`のように、要素の見た目を直接書き換える方法もあります。あらかじめCSSにクラスを用意しておかなくても、その場で色やサイズを指定できます。

ただし、「オンかオフか」のような状態の切り替えには、`classList`で名前の付いたクラスを付け外しする方が、見た目のルールをCSS側にまとめておけて見通しがよくなります。このコースでは深入りしませんが、`style`という直接操作の道具があることだけ、覚えておいてください。

## このレッスンで分かったこと

- `textContent`への代入で、要素の中身の文字をまるごと書き換えられる
- `classList.add("名前")`はクラスを付ける、`remove("名前")`は外す、`toggle("名前")`は今の状態を見て自動で逆を選ぶ
- `toggle`に第2引数（true/false）を渡すと、今の状態に関係なく強制的に付ける・外すを選べる
- `el.style.プロパティ = 値`で見た目を直接書き換える道具もある。オン/オフの切り替えには`classList`の方が見通しがよい
