## 木の中から、1つだけ選ぶ

前回、`document`が木の入口だと分かりました。ここから先へ進むための道具が`document.querySelector`です。

```js nocheck
document.querySelector(セレクタの文字列)
```

「セレクタ」には、CSSと同じ書き方の文字列を渡します。このコースで使うのは、次の3つの最小限の形だけです。

| 書き方 | 意味 | 例 |
|---|---|---|
| `"p"` | タグ名で探す | `<p>`を探す |
| `"#id名"` | idで探す | `id="greeting"`の要素を探す |
| `".class名"` | classで探す | `class="yatai"`の要素を探す |

見つかった要素の中身の文字を読み書きするには、`textContent`というプロパティを使います。読めば中の文字が返り、代入すればその要素の中身がまるごと書き換わります。

<HtmlRunner
  client:visible
  storageKey="dom-02-a"
  height={160}
  initialHtml={`<p id="greeting">こんばんは</p>
<script>
  const p = document.querySelector("#greeting");
  p.textContent = "いらっしゃい、縁日へ";
</script>`}
/>

ページが読み込まれると、最初にHTMLへ書いた「こんばんは」ではなく、「いらっしゃい、縁日へ」が表示されます。JavaScriptが`document.querySelector("#greeting")`で段落を見つけ、`textContent`への代入で中身を上書きしたからです。

## querySelectorは、最初の1つだけを返す

`querySelector`という名前の通り、これは**1つの要素だけ**を探す道具です。同じタグや同じclassの要素が複数あっても、返ってくるのは文書の中で一番先に現れた1つだけです。

<HtmlRunner
  client:visible
  storageKey="dom-02-b"
  height={220}
  initialHtml={`<ul>
  <li>りんご飴</li>
  <li>金魚すくい</li>
  <li>ラムネ</li>
</ul>
<p id="first-item"></p>
<script>
  const first = document.querySelector("li");
  const result = document.querySelector("#first-item");
  result.textContent = "最初のliは: " + first.textContent;
</script>`}
/>

`<li>`は3つありますが、`document.querySelector("li")`はタグ名だけを見て、一番上の「りんご飴」だけを返します。classでも同じことが起きます。

<HtmlRunner
  client:visible
  storageKey="dom-02-c"
  height={200}
  initialHtml={`<p class="yatai">りんご飴 300円</p>
<p class="yatai">金魚すくい 500円</p>
<p class="yatai">ラムネ 200円</p>
<script>
  const first = document.querySelector(".yatai");
  first.textContent = first.textContent + "（本日の一押し）";
</script>`}
/>

3つとも`class="yatai"`を持っていますが、書き換わるのは一番上の「りんご飴」だけです。「複数まとめて取りたい」場面もいずれ出てきますが、このコースでは`querySelector`——1つだけ選ぶ道具——を中心に進めます。

## 見つからなかったら、null

探している`id`や`class`が文書のどこにも無かったら、`querySelector`はエラーにはならず、**null**を返します。

<HtmlRunner
  client:visible
  storageKey="dom-02-d"
  height={140}
  initialHtml={`<p id="result">確認中...</p>
<script>
  const missing = document.querySelector("#nonexistent");
  const result = document.querySelector("#result");
  if (missing === null) {
    result.textContent = "見つかりませんでした（null）";
  } else {
    result.textContent = "見つかりました";
  }
</script>`}
/>

`#nonexistent`という`id`は、このHTMLのどこにも書かれていません。だから`missing`にはnullが入り、「見つかりませんでした」の方が表示されます。

コース3で、「ない」を表す値がundefinedとnullの2つあると学びました。存在しないオブジェクトのキーはundefinedでしたが、**querySelectorが「見つからなかった」ことを表すときはnull**です。名前は違っても、どちらも「そこには値が無い」という点は同じです。

<Callout type="warn" title="つまずきポイント：nullのプロパティを読もうとすると止まる">
nullは「値が無い」ことを表す値そのものなので、`null.textContent`のようにその先のプロパティを読もうとすると、TypeErrorで止まります。`querySelector`の結果をそのまま使う前に、見つかったかどうかを確かめる習慣が大事です。
</Callout>

<CodeRunner
  client:visible
  storageKey="dom-02-e"
  expectError
  initialCode={`// querySelectorが「見つからなかった」ときは、nullが返ります
const el = null;
el.textContent;`}
/>

実行すると、`TypeError: Cannot read properties of null (reading 'textContent')`というエラーで止まります。「nullの中のtextContentを読もうとしたが、nullには中身が無い」という意味です。

<Callout type="mame" title="安全弁を思い出す: オプショナルチェーン">
コース3で見た`?.`を使うと、`document.querySelector("#nonexistent")?.textContent`のように書けます。見つからなければエラーにならず、undefinedになるだけで済みます。見つかるかどうか自信が無いときの安全弁として、覚えておいて損はありません。
</Callout>

## このレッスンで分かったこと

- `document.querySelector(セレクタ)`は、木の中から条件に合う**最初の1つ**の要素を返す
- セレクタは`"p"`（タグ名）・`"#id名"`・`".class名"`の3つの最小限の書き方を使う
- `textContent`で要素の中身を読み書きできる。代入すると中身がまるごと置き換わる
- 見つからなかったときはエラーではなく**null**が返る。nullのプロパティを読もうとするとTypeErrorで止まる
- 使う前に`=== null`で確かめるか、`?.`で安全に読む
