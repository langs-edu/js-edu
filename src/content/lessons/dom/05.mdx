## 読み手から、文字を受け取る

ここまでは、ボタンを押すという「行動」には応えられても、読み手が書いた「言葉」そのものを受け取ってはいませんでした。それを受け取る入り口が`<input>`です。

```html
<input id="name-input" type="text" placeholder="お名前">
```

`<input>`は`<p>`と違い、終了タグを書きません。読み手が打ち込んだ文字は、`.value`というプロパティで読み取れます。

<HtmlRunner
  client:visible
  storageKey="dom-05-a"
  height={180}
  initialHtml={`<input id="name-input" type="text" placeholder="お名前">
<button id="greet-btn">呼びかける</button>
<p id="greet-result"></p>
<script>
  const input = document.querySelector("#name-input");
  const button = document.querySelector("#greet-btn");
  const result = document.querySelector("#greet-result");
  button.addEventListener("click", () => {
    result.textContent = input.value + "さん、いらっしゃい";
  });
</script>`}
/>

実際に自分の名前を入力欄に打ち込んで、「呼びかける」ボタンを押してみてください。入力した文字の後ろに「さん、いらっしゃい」が続いて表示されます。`input.value`が、その瞬間に入力欄に入っている文字を返しています。

## .valueは、いつでも文字列

ここで、大事な誠実な事実があります。**`.value`は、`type`が何であっても常に文字列**です。数字だけを入れるつもりの`<input>`であっても同じです。

<CodeRunner
  client:visible
  storageKey="dom-05-b"
  initialCode={`// input.value は、どんな type の input でも常に文字列です
const aValue = "3";  // 例えば <input value="3"> から読み取った値
const bValue = "4";  // 例えば <input value="4"> から読み取った値
console.log(aValue + bValue);
console.log(Number(aValue) + Number(bValue));`}
/>

実行すると、`aValue + bValue`は数の足し算にはならず、`"34"`という文字列の連結になります。両方とも文字列だからです。数として計算したいときは、`Number()`で変換してから足す必要があります。`Number(aValue) + Number(bValue)`なら、期待通り`7`になります。

同じことを、実際のフォームでも確かめてみましょう。

<HtmlRunner
  client:visible
  storageKey="dom-05-c"
  height={200}
  initialHtml={`<input id="num-a" type="text" value="3">
<input id="num-b" type="text" value="4">
<button id="add-btn">たす</button>
<p id="sum-result"></p>
<script>
  const a = document.querySelector("#num-a");
  const b = document.querySelector("#num-b");
  const button = document.querySelector("#add-btn");
  const result = document.querySelector("#sum-result");
  button.addEventListener("click", () => {
    const total = Number(a.value) + Number(b.value);
    result.textContent = "合計: " + total;
  });
</script>`}
/>

そのままボタンを押すと、`3`と`4`が`Number()`で数値に変換されてから足され、「合計: 7」と表示されます。入力欄の数字を書き換えて、もう一度押してみてください。

<Callout type="warn" title="つまずきポイント：Number()を忘れると文字列がつながるだけ">
`Number(a.value) + Number(b.value)`の`Number()`を省いて`a.value + b.value`と書くと、エラーにはならずに`"34"`のような文字列がそのままつながります。JSは「止まって教えてくれるとは限らない読み手」でした。数として使うつもりなら、`Number()`で変換したかどうかを自分で確かめる習慣が要ります。
</Callout>

## 空チェックという、簡単なバリデーション

何も入力されていないまま送信されたら困る、という場面もよくあります。`.value`が空文字列`""`かどうかを`if`で確かめれば、簡単なチェックができます。

<HtmlRunner
  client:visible
  storageKey="dom-05-d"
  height={180}
  initialHtml={`<input id="wish-input" type="text" placeholder="お願いごと">
<button id="wish-btn">奉納する</button>
<p id="wish-result"></p>
<script>
  const input = document.querySelector("#wish-input");
  const button = document.querySelector("#wish-btn");
  const result = document.querySelector("#wish-result");
  button.addEventListener("click", () => {
    if (input.value === "") {
      result.textContent = "お願いごとを書いてください";
    } else {
      result.textContent = "「" + input.value + "」を奉納しました";
    }
  });
</script>`}
/>

何も書かずに「奉納する」を押すと「お願いごとを書いてください」と表示されます。何か書いてから押すと、その内容が「」で囲まれて表示されます。`input.value === ""`という比較1つで、「まだ何も書かれていない」ことを見分けられます。

## このレッスンで分かったこと

- `<input>`は終了タグの無い要素。読み手が打ち込んだ文字は`.value`で読み取れる
- `.value`は、`type`に関わらず**常に文字列**。数として計算するには`Number()`で変換する
- `Number()`を忘れると、エラーにならずに文字列の連結になってしまう
- `.value === ""`で、まだ何も入力されていないかどうかを確かめられる
