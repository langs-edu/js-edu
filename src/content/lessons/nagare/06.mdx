## 名前は、どこまで見える？

前回、関数の中で`const buki = "スーパーボール";`のような変数を作りました。この`buki`という名前は、関数の外からも使えるのでしょうか。`if`や`for`の`{ }`の中で作った変数はどうでしょう。名前が「どこからどこまで見えるか」という範囲のことを**スコープ**と呼びます。

## ブロックスコープ——{}の中で作った名前は、外から見えない

`let`・`const`で作った変数は、それを囲む`{ }`の中でだけ存在します。

<CodeRunner
  client:visible
  storageKey="nagare-06-a"
  expectError
  initialCode={`if (true) {
  const himitsu = "屋台の裏メニュー";
  console.log(himitsu);
}

console.log(himitsu);`}
/>

```console
屋台の裏メニュー
ReferenceError: himitsu is not defined
```

`if`の`{ }`の中で宣言した`himitsu`は、その`{ }`を出た瞬間に存在しなくなります。`{ }`の外で`console.log(himitsu)`と書くと、`himitsu`という名前そのものが見つからず`ReferenceError`になります。この「`{ }`が名前の見える範囲を区切る」性質を**ブロックスコープ**と呼びます。

<Callout type="note" title="宣言と代入は別物">
ブロックスコープが区切るのは、あくまで**新しく`let`・`const`で宣言した名前**です。すでに外で宣言されている変数に、`{ }`の中から値を代入するだけなら話は別です。
</Callout>

<CodeRunner
  client:visible
  storageKey="nagare-06-b"
  initialCode={`let kyakusuu = 0;

if (true) {
  kyakusuu = kyakusuu + 10;
}

console.log(kyakusuu);`}
/>

```console
10
```

ここでは`if`の`{ }`の中に`let`も`const`もありません。外で宣言済みの`kyakusuu`に、`{ }`の中から代入し直しているだけです。だから`{ }`を出たあとも、変更はちゃんと反映されています。

## 関数スコープ——関数の中で作った名前は、外から見えない

関数の`{ }`も、同じルールです。

<CodeRunner
  client:visible
  storageKey="nagare-06-c"
  expectError
  initialCode={`function junbi() {
  const buki = "スーパーボール";
  console.log(buki);
}

junbi();
console.log(buki);`}
/>

```console
スーパーボール
ReferenceError: buki is not defined
```

`junbi`関数の中で作った`buki`は、関数の外からは見えません。関数を呼び出した後でも、その中だけで生まれて消える名前です。

## 外の変数を、関数の中から変更する

先ほどの`if`と同じように、関数もすでにある外側の変数を読んだり書き換えたりできます。関数の中で改めて`let`・`const`で宣言していなければ、それは新しい変数ではなく「外の変数そのもの」です。

<CodeRunner
  client:visible
  storageKey="nagare-06-d"
  initialCode={`let kyakusuu = 0;

function kyakuGaKuru() {
  kyakusuu = kyakusuu + 1;
}

kyakuGaKuru();
kyakuGaKuru();
console.log(kyakusuu);`}
/>

```console
2
```

`kyakuGaKuru`を呼び出すたびに、外の`kyakusuu`が1ずつ増えています。関数の中に`let kyakusuu`という宣言がないので、これは新しい変数ではなく、外の`kyakusuu`への代入です。

## シャドーイング——同じ名前で、内側が外側を覆い隠す

もし関数の中で、外と同じ名前を`let`・`const`で改めて宣言したらどうなるでしょうか。

<CodeRunner
  client:visible
  storageKey="nagare-06-e"
  initialCode={`let namae = "コウタ";

function yobu() {
  let namae = "あかり";
  console.log(namae);
}

yobu();
console.log(namae);`}
/>

```console
あかり
コウタ
```

関数の中の`let namae = "あかり";`は、外の`namae`とは別の、新しい変数です。関数の中では、この内側の`namae`が優先され、外の`namae`は一時的に「覆い隠され」ます。これを**シャドーイング**と呼びます。関数を抜けると内側の`namae`は消えるので、外の`namae`は`"コウタ"`のまま変わっていません。

<Callout type="warn" title="つまずきポイント：うっかりシャドーイングして、更新のつもりが更新できていない">
「外の変数を増やしたい」と思って関数の中に書いたつもりが、うっかり`let`や`const`をつけてしまうと、それは更新ではなく**別の変数の新規作成**になります。エラーにはならず、ただ外の変数が変わらないまま——静かに意図と違う動きをするので、見つけにくいバグです。外の変数を更新したいときは、`let`・`const`をつけずに`名前 = 値;`とだけ書いてください。
</Callout>

## このレッスンで分かったこと

- `{ }`の中で`let`・`const`で宣言した変数は、その`{ }`の外からは見えない（ブロックスコープ）。これはif・for・関数のどの`{ }`でも同じ
- 外ですでに宣言済みの変数に`{ }`の中から代入するだけなら、`{ }`を出たあとも変更は残る
- 関数の中で外と同じ名前を`let`・`const`で改めて宣言すると、それは別の新しい変数になり、外の変数を覆い隠す（シャドーイング）
- 外の変数を更新したいときは、`let`・`const`をつけずに`名前 = 値;`とだけ書く
