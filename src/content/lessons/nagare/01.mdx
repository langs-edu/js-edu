## プログラムに、判断を教える

前のコースの最後で、こんな予告をしていました。「道具がそろいました。次のコースでは、プログラムに『判断』を教えます」。ここから、その始まりです。

ここまで書いてきたプログラムは、上から下へ、書いた文を全部そのまま実行するだけでした。でも実際のプログラムは、状況によって振る舞いを変えたくなります。「お客さんが多ければ、呼び込みの声を大きくする。在庫がなければ、売り切れと伝える」——こんな分かれ道を書くための文が、今日から学ぶ`if`です。

## 比較して、真偽値を作る

`if`が判断の材料にするのは、`true`か`false`のどちらかです。コース1で出会った`boolean`型を覚えていますか。分かれ道を書く前に、`true`と`false`をどうやって手に入れるかを見ておきましょう。

<CodeRunner
  client:visible
  storageKey="nagare-01-a"
  initialCode={`const ondo = 32;
console.log(ondo === 32);
console.log(ondo !== 32);
console.log(ondo > 30);
console.log(ondo < 30);
console.log(ondo >= 32);
console.log(ondo <= 31);`}
/>

実行すると、下の出力欄にこう表示されます。

```console
true
false
true
false
true
false
```

| 演算子 | 意味 |
|---|---|
| `===` | 等しい |
| `!==` | 等しくない |
| `<` | より小さい |
| `>` | より大きい |
| `<=` | 以下 |
| `>=` | 以上 |

`===`は「等しいか」を調べる演算子です。`=`（1つ）が代入だったのに対し、`===`は比較——見た目も意味も違いますが、最初は混同しやすいので注意してください。比較の結果は、いつも`boolean`型の値、`true`か`false`のどちらか一方です。

## if文——条件がtrueなら実行する

比較の結果を使って、実際に分かれ道を書いてみます。

<CodeRunner
  client:visible
  storageKey="nagare-01-b"
  initialCode={`const kyakusuu = 120;

if (kyakusuu > 100) {
  console.log("たくさんのお客さんです");
}

console.log("今日の売上をまとめます");`}
/>

```console
たくさんのお客さんです
今日の売上をまとめます
```

`if (条件) { ... }`の`{ }`の中だけが、「条件が`true`のときに実行される」範囲です。試しに`kyakusuu`を`60`に書き換えて実行してみてください。条件が`false`になるので、`{ }`の中身は実行されず、次のように`今日の売上をまとめます`だけが表示されるはずです。

```console
今日の売上をまとめます
```

## {}が作るブロックの範囲

JavaScriptでは、`if`が実行する範囲を決めているのは**インデント（字下げ）ではなく、`{ }`そのもの**です。インデントはあくまで人間が読みやすくするための習慣であって、書かなくても動いてしまいます。これは、字下げそのものが文法になっている言語とは違う、JSならではの性格です。

だからこそ、`{ }`を省略すると、見た目とは違う動きをする事故が起きます。

```js title="MisleadingIndent.js"
const kyakusuu = 60;

if (kyakusuu > 100)
  console.log("たくさんのお客さんです");
  console.log("列の整理をお願いします");
```

```console
列の整理をお願いします
```

見た目では、2行とも`if`の中に字下げされていて、両方とも「お客さんが多いときだけ」実行されそうに見えます。でも`{ }`がないので、`if`の対象になるのは**次の1文だけ**——`console.log("たくさんのお客さんです");`だけです。`kyakusuu`が`60`で条件が`false`でも、2行目の`console.log("列の整理をお願いします")`は`if`とは無関係な、いつも実行される文として動いてしまいます。

<Callout type="warn" title="つまずきポイント：{}を省略した1文ルール">
`if (条件)`のあとに`{ }`がないとき、`if`が制御するのは直後の1文だけです。2文以上を条件つきにしたいときは、必ず`{ }`で囲んでください。**このサイトでは、`if`には常に`{ }`をつけて書きます。** 1文だけでも省略しない——事故を防ぐいちばん簡単な習慣です。
</Callout>

## else——それ以外なら

`if`が`false`だったときの行き先も書けます。

<CodeRunner
  client:visible
  storageKey="nagare-01-c"
  initialCode={`const zaiko = 0;

if (zaiko > 0) {
  console.log("りんご飴、まだあります");
} else {
  console.log("りんご飴は売り切れです");
}`}
/>

```console
りんご飴は売り切れです
```

`zaiko`が`0`なので`zaiko > 0`は`false`、`else`の`{ }`が実行されました。`if`と`else`、どちらか一方だけが必ず実行されます。

## else if——複数の分かれ道

3つ以上に分けたいときは、`else if`を間にはさみます。

<CodeRunner
  client:visible
  storageKey="nagare-01-d"
  initialCode={`const nenrei = 10;
let nyuujouryou;

if (nenrei < 6) {
  nyuujouryou = 0;
} else if (nenrei < 13) {
  nyuujouryou = 300;
} else {
  nyuujouryou = 800;
}

console.log(\`\${nenrei}さいの入場料は\${nyuujouryou}円です\`);`}
/>

```console
10さいの入場料は300円です
```

上から順に条件が調べられ、最初に`true`になったブロックだけが実行されて、残りは調べられません。`nenrei`は`10`なので`nenrei < 6`は`false`、`nenrei < 13`は`true`——ここで確定し、`nyuujouryou = 800`は調べられもしません。`else if`はいくつでも重ねられます。

## && ・ || ・ !——条件を組み合わせる

複数の条件をまとめて1つの判断にしたいときは、`&&`（かつ）・`||`（または）・`!`（〜でない）を使います。

<CodeRunner
  client:visible
  storageKey="nagare-01-e"
  initialCode={`const isGakusei = true;
const isHeijitsu = false;

if (isGakusei && isHeijitsu) {
  console.log("学生平日割引を適用します");
} else {
  console.log("通常料金です");
}

console.log(!isHeijitsu);
console.log(isGakusei || isHeijitsu);`}
/>

```console
通常料金です
true
true
```

| 演算子 | 意味 |
|---|---|
| `&&` | 両方が`true`のときだけ全体が`true` |
| `\|\|` | どちらか一方でも`true`なら全体が`true` |
| `!` | `true`と`false`をひっくり返す |

`isHeijitsu`が`false`なので`isGakusei && isHeijitsu`全体は`false`になり、`else`が実行されました。`!isHeijitsu`は`false`をひっくり返して`true`、`isGakusei || isHeijitsu`はどちらか一方（`isGakusei`）が`true`なので全体も`true`です。

## このレッスンで分かったこと

- 比較演算子（`===` `!==` `<` `>` `<=` `>=`）は`boolean`型の値を作る
- `if (条件) { ... }`の`{ }`の中だけが、条件が`true`のときに実行される
- `{ }`はJSでは省略できてしまう。省略すると、次の1文だけが`if`の対象になり、見た目と違う動きの事故が起きる。このサイトでは常に`{ }`をつける
- `else`は`false`のときの行き先、`else if`は3つ以上に分かれ道を増やす。上から順に調べ、最初に`true`になったところだけが実行される
- `&&`・`||`・`!`で条件を組み合わせられる
