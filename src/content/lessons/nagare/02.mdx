## ===という約束

前回、比較に`===`を使いました。ここまでは特に説明せずに使ってきましたが、実は`===`という書き方には理由があります。JavaScriptには、等しいかどうかを調べる演算子が**2種類**あるのです。このレッスンでは、その正体を確かめます。

## ==という、もうひとつの等号

`===`より前から存在するのが、`==`です。試してみましょう。

<CodeRunner
  client:visible
  storageKey="nagare-02-a"
  initialCode={`console.log("1" == 1);
console.log("1" === 1);
console.log(0 == false);
console.log(0 === false);`}
/>

```console
true
false
true
false
```

`"1"`は文字列、`1`は数値です。型がまったく違うのに、`==`は`true`を返しました。`==`は、比べる前にこっそり型を変換してから比べる演算子だからです。`0 == false`も同じ理由で`true`になります——数値の`0`と真偽値の`false`という、別の型どうしです。

一方`===`は、型変換をいっさいしません。**型も値も両方一致していないと`true`にならない。** だから`"1" === 1`は`false`です。

<Callout type="warn" title="つまずきポイント：==は読めるとよいが、書かない">
`==`は今でも動きますし、古いコードでは今も見かけます。読めることは大事です。でも、これから自分で書くときは`===`を使ってください。`==`が行う型変換のルールは複雑で、`"" == 0`のように直感に反する`true`も生まれます。「型がゆるく変換されて比べられる」という性質そのものが、意図しないバグの温床になりやすいのです。**このサイトでは、常に`===`（と`!==`）だけを書きます。**
</Callout>

`==`が生まれた歴史と、なぜ今も残っているのかは、[🏮 屋台のこぼれ話「==と===の物語」](../../../yatai/09-equality/)で詳しく扱っています。

## truthyとfalsy——trueでもfalseでもない値が、ifの中で化ける

`if`の`( )`の中には、`boolean`型の値しか置けないと思うかもしれません。でも実際には、数値でも文字列でも、何でも置けます。

<CodeRunner
  client:visible
  storageKey="nagare-02-b"
  initialCode={`if (0) {
  console.log("実行される");
} else {
  console.log("実行されない");
}

if ("") {
  console.log("実行される");
} else {
  console.log("実行されない");
}

if ("0") {
  console.log("実行される");
} else {
  console.log("実行されない");
}`}
/>

```console
実行されない
実行されない
実行される
```

`if`は`( )`の中身が`boolean`でなければ、自動的に`true`か`false`に変換してから判断します。`0`は`false`扱い、`""`（空文字列）も`false`扱い。でも`"0"`——`0`という文字が1文字入った文字列——は空ではないので`true`扱いです。数字の`0`と、文字列の`"0"`とで結果が変わる、これも型がからむ落とし穴です。

`false`として扱われる値は、次の5つだけです。これを**falsy**と呼びます。

<CodeRunner
  client:visible
  storageKey="nagare-02-c"
  initialCode={`console.log(Boolean(0));
console.log(Boolean(""));
console.log(Boolean(null));
console.log(Boolean(undefined));
console.log(Boolean(NaN));`}
/>

```console
false
false
false
false
false
```

`Boolean()`は、値を強制的に`true`か`false`に変換する関数です。falsyな値は`0`・`""`・`null`・`undefined`・`NaN`の5つだけ——**それ以外の値はすべてtruthy（true扱い）**です。`"0"`も`" "`（スペース1つ）も、空でない文字列だからtruthyになります。

## if (name)という慣習

truthy・falsyを知ると、こんな書き方が読めるようになります。

<CodeRunner
  client:visible
  storageKey="nagare-02-d"
  initialCode={`const namae = "";

if (namae) {
  console.log(\`こんにちは、\${namae}さん\`);
} else {
  console.log("名前が入力されていません");
}`}
/>

```console
名前が入力されていません
```

`if (namae)`は、`namae === ""`のような比較を書かなくても、「`namae`に何か中身があるかどうか」を短く判定できます。`namae`を`"コウタ"`に書き換えて実行すると、今度は`こんにちは、コウタさん`と表示されます。中身が入っているかどうかをざっくり調べたいとき、JSのコードでよく見かける書き方です。

<Callout type="mame" title="NaN === NaN は false">
falsyな値の中でも`NaN`だけは、ちょっと変わった性質があります。`NaN === NaN`は`false`になるのです。「Not a Number」自身とすら等しくない——コース1の06で出会った`NaN`の奇妙さは、こんなところにも顔を出します。値が`NaN`かどうかを調べたいときは`===`ではなく、専用の`Number.isNaN()`という関数を使います（名前だけ、ここでは覚えなくて大丈夫です）。
</Callout>

## このレッスンで分かったこと

- `==`は型を変換してから比べる古い演算子。`"1" == 1`は`true`になる。読めるとよいが、自分では書かない
- `===`は型も値も両方一致していないと`true`にならない。このサイトでは常に`===`（と`!==`）を使う
- `if`の`( )`の中身は、`boolean`でなくても自動的にtrue/falseへ変換される。falsyなのは`0`・`""`・`null`・`undefined`・`NaN`の5つだけで、それ以外はすべてtruthy
- `if (name)`は「中身があるかどうか」を短く判定する、JSでよく見る慣習
- `NaN === NaN`は`false`になる。自分自身とすら一致しない、`NaN`だけの奇妙な性質
