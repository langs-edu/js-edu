## 数えるくりかえしには、専用の書き方がある

前回の`while`で、提灯を5個つけるプログラムを書きました。もう一度見てみます。

```js title="Lantern.js"
let count = 0;
const total = 5;

while (count < total) {
  count = count + 1;
  console.log(`${count}個目の提灯をつけました`);
}
```

「変数を用意する」「条件を調べる」「変数を更新する」の3つが、コードのあちこちに散らばっています。`count = count + 1;`を書き忘れると無限ループになる、という危険も前回見た通りです。「決まった回数だけくりかえす」という、とてもよくある場面には、この3つを1か所にまとめて書ける専用の文があります。`for`です。

## for文の基本

<CodeRunner
  client:visible
  storageKey="nagare-04-a"
  initialCode={`for (let i = 1; i <= 5; i++) {
  console.log(\`\${i}個目の提灯をつけました\`);
}`}
/>

```console
1個目の提灯をつけました
2個目の提灯をつけました
3個目の提灯をつけました
4個目の提灯をつけました
5個目の提灯をつけました
```

`for (初期化; 条件; 更新) { ... }`という形です。3つの部分は、それぞれこう対応しています。

| 部分 | このコードでの中身 | 役割 |
|---|---|---|
| 初期化 | `let i = 1` | ループが始まる前に1回だけ実行される |
| 条件 | `i <= 5` | 毎回チェックされ、trueの間だけくりかえす |
| 更新 | `i++` | 1回のくりかえしが終わるたびに実行される |

`while`では別々の場所に書いていた3つが、`for`の`( )`の中に1行でまとまりました。「更新を書き忘れて無限ループになる」という事故も、`for`なら`( )`を見ればひと目で気づけます。

`i++`は`i = i + 1`の短縮形です。`i`を1増やすだけの、とてもよく使う操作なので専用の書き方が用意されています。同じように`i--`は`i = i - 1`の短縮形です。

`while`と`for`、どちらを使うか迷ったら、こう考えてください。「何回くりかえすか、始める前から決まっている」なら`for`。「何回くりかえすかは分からないけれど、ある条件を満たすまで続けたい」なら`while`。前回の数当てゲームの下準備で見た「7の倍数が見つかるまで」のようなくりかえしは、何回で見つかるか事前には分からないので`while`が向いています。一方、今日の九九のように「1から9まで」と最初から回数が決まっているくりかえしは`for`が向いています。どちらで書いても動くことは多いですが、意図が伝わりやすいほうを選ぶと、あとで読み返したときに親切です。

## forを入れ子にする——ネスト

`for`の`{ }`の中に、もう1つ`for`を書けます。これを**ネスト**（入れ子）と呼びます。

<CodeRunner
  client:visible
  storageKey="nagare-04-c"
  initialCode={`for (let dan = 2; dan <= 4; dan++) {
  for (let i = 1; i <= 3; i++) {
    console.log(\`\${dan} x \${i} = \${dan * i}\`);
  }
}`}
/>

```console
2 x 1 = 2
2 x 2 = 4
2 x 3 = 6
3 x 1 = 3
3 x 2 = 6
3 x 3 = 9
4 x 1 = 4
4 x 2 = 8
4 x 3 = 12
```

外側の`for (let dan = 2; dan <= 4; dan++)`が1回進むたびに、内側の`for (let i = 1; i <= 3; i++)`が最初から最後まで、まるごと実行されます。`dan`が`2`のとき内側が3回、`dan`が`3`のときまた内側が3回……という具合に、外側と内側の回数がかけ算されます。ここでは外側3回×内側3回で、合計9行が表示されました。

<Callout type="warn" title="つまずきポイント：内側と外側の変数名を同じにしない">
ネストしたループでは、内側と外側で別の変数名（ここでは`dan`と`i`）を使ってください。もし両方`i`という名前にしてしまうと、内側の`for`が外側の`i`を上書きしてしまい、外側のくりかえしが狂います。名前がどこまで見えるか（スコープ）については、このコースの後半で詳しく扱います。
</Callout>

## このレッスンで分かったこと

- `for (初期化; 条件; 更新) { ... }`は、初期化・条件・更新を1か所にまとめて書ける、数え上げるくりかえし専用の文
- `i++`は`i = i + 1`の短縮形。`i--`も同様に`i = i - 1`の短縮形
- `for`の中にもう1つ`for`を書くことをネスト（入れ子）と呼ぶ。外側が1回進むごとに、内側が最初から最後まで実行される
- ネストしたループでは、内側と外側で違う変数名を使う
