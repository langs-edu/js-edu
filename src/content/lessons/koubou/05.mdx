## hondana.jsが、大きくなりすぎた

いまの`hondana.js`には、4つの役割が同居しています。

- 画面の要素を取ってくる
- フォームやボタンのイベントに応える
- 本を検証する(`createBook`)
- 本を検索する(`searchBooks`)

`searchBooks`が正しく動くかを確かめたいだけなのに、テストしようとすると`document.querySelector`が失敗したり、`localStorage`が必要だったりと、**画面という別の関心事が、いつもついてきてしまいます。** 今日は、動くコードを壊さずに、この2つを引き離します。

## 分ける方針

- **`storage.js`** — すでに、`localStorage`の読み書きだけをする係になっています(レッスン2)
- **`logic.js`** — 検索・集計・検証の**計算だけ**をする係。DOMにも`localStorage`にも一切触れません。今日、新しく作ります
- **`hondana.js`** — この2つを呼び出して組み合わせる、画面の配線係として残ります

`logic.js`がDOMに触れなくなれば、テストは`{ title, author, rating, review }`のようなプレーンオブジェクトの配列を直接渡すだけで済みます。画面の用意も後片付けも要りません。

## node:testの、最初の一歩

Node.js 22には、**追加のインストールなしで**使えるテストの仕組み、`node:test`が標準で入っています。`test`という関数と、`node:assert/strict`の`assert`を組み合わせて使います。

<CodeRunner
  client:visible
  storageKey="koubou-05-a"
  initialCode={`import { test } from 'node:test';
import assert from 'node:assert/strict';

test('1 + 1 は 2', () => {
  assert.equal(1 + 1, 2);
});

test('文字列を連結できる', () => {
  assert.equal("縁" + "日", "縁日");
});`}
/>

`test(名前, () => { ... })`で、1つのテストを登録します。中の関数で`assert.equal(実際の値, 期待する値)`のように書き、値が一致していなければ、そこで失敗を報告します。実行すると、出力欄に**TAP**という、テスト結果の共通フォーマットで結果が表示されます。`ok 1 - 1 + 1 は 2`のような行が、1つのテストの結果です。

## 数値の範囲チェックに潜む、NaNの落とし穴

`logic.js`を書く前に、1つ確かめておきたいことがあります。評価が1〜5の範囲内かどうかを調べる、小さな関数を書いてみます。

<CodeRunner
  client:visible
  storageKey="koubou-05-b"
  expectError
  initialCode={`import { test } from 'node:test';
import assert from 'node:assert/strict';

function isValidRating(rating) {
  return !(rating < 1 || rating > 5);
}

test('範囲内ならtrue', () => {
  assert.equal(isValidRating(3), true);
});

test('範囲外ならfalse', () => {
  assert.equal(isValidRating(0), false);
});

test('数字でなければfalseになってほしい', () => {
  assert.equal(isValidRating(Number('abc')), false);
});`}
/>

3つ目のテストが失敗します。`Number('abc')`は`NaN`という値になりますが、`NaN < 1`も`NaN > 5`も、**どちらも`false`です。** `NaN`はどんな数と比べても`true`になりません。だから`!(false || false)`は`true`になり、`isValidRating(NaN)`は`true`を返してしまいます——コース1で出会った、「NaNはエラーにならず静かに流れていく」という性質が、ここにも表れています。

比べ方を、次のように変えると直ります。

<CodeRunner
  client:visible
  storageKey="koubou-05-c"
  initialCode={`import { test } from 'node:test';
import assert from 'node:assert/strict';

function isValidRating(rating) {
  return rating >= 1 && rating <= 5;
}

test('範囲内ならtrue', () => {
  assert.equal(isValidRating(3), true);
});

test('範囲外ならfalse', () => {
  assert.equal(isValidRating(0), false);
});

test('数字でなければfalseになってほしい', () => {
  assert.equal(isValidRating(Number('abc')), false);
});`}
/>

`rating >= 1 && rating <= 5`という形なら、`NaN >= 1`も`NaN <= 5`も`false`なので、`false && false`はやはり`false`になり、`NaN`は正しく「範囲外」として扱われます。**「〜より小さいか大きい」ではなく「〜以上かつ〜以下」の形で書く**——この小さな書き換えだけで、NaNの落とし穴を避けられます。`logic.js`の`createBook`は、最初からこの形で書くことにします。

## logic.jsを取り出す

検索・集計・検証を、`hondana.js`から`logic.js`に移します。評価の平均と著者別冊数は、これまでのhondanaにはまだなかった、新しい機能です。

```js title="logic.js"
// logic.js
// 本の検証・検索・集計だけを行う。DOMにもlocalStorageにも触れない。

export function createBook(title, author, rating, review) {
  if (!title) {
    throw new Error("タイトルは必須です");
  }
  if (!author) {
    throw new Error("著者は必須です");
  }
  if (!(rating >= 1 && rating <= 5)) {
    throw new Error(`評価は1から5で入力してください: ${rating}`);
  }
  return { title, author, rating, review };
}

export function searchBooks(books, keyword) {
  return books.filter(
    (book) => book.title.includes(keyword) || book.author.includes(keyword)
  );
}

export function averageRating(books) {
  if (books.length === 0) {
    return 0;
  }
  let total = 0;
  for (const book of books) {
    total += book.rating;
  }
  return total / books.length;
}

export function countByAuthor(books) {
  const counts = {};
  for (const book of books) {
    counts[book.author] = (counts[book.author] ?? 0) + 1;
  }
  return counts;
}
```

`averageRating`と`countByAuthor`は、どちらもfor-ofで書いています。配列には`reduce`という集計向けのメソッドもあります(コース4で名前だけ触れました)が、hondanaでは読みやすさを優先して、コース2・3から使ってきたfor-ofのままにしています。`countByAuthor`の`counts[book.author] = (counts[book.author] ?? 0) + 1;`は、コース3の単語カウンタ(レッスン7)で書いたのと、まったく同じ形です。

## storage.jsも、テストしやすく整理する

`storage.js`の`loadBooks`は、「JSON文字列を本の配列に戻す」という部分と、「`localStorage`から読む」という部分が混ざっていました。前者だけを、`parseStoredBooks`という独立した関数に切り出します。

```js title="storage.js"
// storage.js
// 本の一覧を、ブラウザのlocalStorageへ保存・読み込みする係。

const STORAGE_KEY = "hondana-books";

export function parseStoredBooks(json) {
  if (json === null) {
    return [];
  }
  const parsed = JSON.parse(json);
  return Array.isArray(parsed) ? parsed : [];
}

export function loadBooks() {
  return parseStoredBooks(localStorage.getItem(STORAGE_KEY));
}

export function saveBooks(books) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(books));
}
```

`Array.isArray(parsed) ? parsed : []`という行も、新しく足しました。もし開発者ツールなどから`localStorage`の中身を直接書き換えられて、配列ではない値(たとえば`{"broken":true}`)が保存されていたら、そのままでは`books.length`や`for...of`が期待どおりに動きません。**配列であることまで確かめてから返す**、という一段階の守りです。`parseStoredBooks`は`localStorage`に一切触れないので、`loadBooks`から切り離して単体でテストできます。

## hondana.jsは、importを差し替えるだけ

`createBook`と`searchBooks`の中身は`hondana.js`から消え、かわりに`logic.js`からimportします。それ以外の画面の配線は変わりません。加えて、評価の平均と著者別冊数を表示する`renderStats`を1つ追加します。

```js nocheck
// これまで: hondana.jsの中に検証・検索まで書いていた
function createBook(title, author, rating, review) { /* ... */ }
function searchBooks(books, keyword) { /* ... */ }

// これから: logic.jsとstorage.jsに任せる
import { loadBooks, saveBooks } from './storage.js';
import { createBook, searchBooks, averageRating, countByAuthor } from './logic.js';
```

```js title="hondana.js" nocheck
import { loadBooks, saveBooks } from './storage.js';
import { createBook, searchBooks, averageRating, countByAuthor } from './logic.js';

let books = loadBooks();

const listEl = document.querySelector('#book-list');
const formEl = document.querySelector('#add-form');
const titleInput = document.querySelector('#title-input');
const authorInput = document.querySelector('#author-input');
const ratingInput = document.querySelector('#rating-input');
const reviewInput = document.querySelector('#review-input');
const errorEl = document.querySelector('#error');
const searchInput = document.querySelector('#search-input');
const statsEl = document.querySelector('#stats');

function render(list) {
  listEl.textContent = "";

  if (list.length === 0) {
    const empty = document.createElement('li');
    empty.textContent = "まだ本が登録されていません。";
    listEl.appendChild(empty);
  } else {
    for (const book of list) {
      const li = document.createElement('li');

      const heading = document.createElement('strong');
      heading.textContent = `${book.title} / ${book.author} (評価${book.rating})`;

      const review = document.createElement('p');
      review.textContent = book.review;

      const deleteButton = document.createElement('button');
      deleteButton.textContent = "削除";
      deleteButton.addEventListener('click', () => {
        books = books.filter((b) => b !== book);
        saveBooks(books);
        render(books);
      });

      li.appendChild(heading);
      li.appendChild(review);
      li.appendChild(deleteButton);
      listEl.appendChild(li);
    }
  }

  renderStats();
}

function renderStats() {
  if (books.length === 0) {
    statsEl.textContent = "";
    return;
  }

  const average = averageRating(books);
  const counts = countByAuthor(books);
  const authorNames = Object.keys(counts);

  const lines = [];
  for (const author of authorNames) {
    lines.push(`${author}: ${counts[author]}冊`);
  }

  statsEl.textContent = `${books.length}冊 / 平均評価 ${average.toFixed(2)} / ${lines.join("、")}`;
}

formEl.addEventListener('submit', (event) => {
  event.preventDefault();
  errorEl.textContent = "";

  try {
    const book = createBook(
      titleInput.value,
      authorInput.value,
      Number(ratingInput.value),
      reviewInput.value
    );
    books = [...books, book];
    saveBooks(books);
    render(books);

    titleInput.value = "";
    authorInput.value = "";
    ratingInput.value = "";
    reviewInput.value = "";
  } catch (error) {
    errorEl.textContent = error.message;
  }
});

searchInput.addEventListener('input', () => {
  const keyword = searchInput.value;
  render(keyword === "" ? books : searchBooks(books, keyword));
});

render(books);
```

`index.html`には、集計を表示する`<p id="stats">`を1つ足すだけです。

```html
<input id="search-input" type="search" placeholder="検索">
<p id="stats"></p>

<ul id="book-list"></ul>
```

`renderStats`の`Object.keys(counts)`は、`countByAuthor`が返したオブジェクトから、著者名だけを配列として取り出すメソッドです。取り出した配列を、これまでどおりfor-ofで歩きます。`.toFixed(2)`は、数値を小数点以下2桁までの文字列にするメソッドで、`4`なら`"4.00"`になります。

<Callout type="note" title="このファイルの検証について">
`hondana.js`は`storage.js`・`logic.js`と組み合わさってはじめて成立するので、CodeRunnerでは単体実行できません。`logic.js`と`storage.js`は、それぞれ単体でNode.js 22の構文チェックを通しています(このあとのテストで、動きそのものも確認します)。
</Callout>

## logic.jsとstorage.jsを、node:testで固める

`tests`というフォルダを作り、その中に`logic.test.mjs`というテストファイルを置きます。

```js title="tests/logic.test.mjs" nocheck
import { test } from 'node:test';
import assert from 'node:assert/strict';

import { createBook, searchBooks, averageRating, countByAuthor } from '../logic.js';
import { parseStoredBooks } from '../storage.js';

const shelf = [
  createBook('三体', '劉慈欣', 5, 'スケールがすごい'),
  createBook('三体0', '劉慈欣', 4, '続編も良い'),
  createBook('コンビニ人間', '村田沙耶香', 3, '静かな衝撃'),
];

test('タイトルに一致する本を探せる', () => {
  const found = searchBooks(shelf, '三体');
  assert.equal(found.length, 2);
});

test('著者に一致する本を探せる', () => {
  const found = searchBooks(shelf, '村田沙耶香');
  assert.equal(found.length, 1);
  assert.equal(found[0].title, 'コンビニ人間');
});

test('一致しない語は空の一覧になる', () => {
  const found = searchBooks(shelf, '存在しない語');
  assert.equal(found.length, 0);
});

test('平均評価を計算できる', () => {
  assert.equal(averageRating(shelf), 4);
});

test('本が空なら平均評価は0', () => {
  assert.equal(averageRating([]), 0);
});

test('著者ごとの冊数を数えられる', () => {
  const counts = countByAuthor(shelf);
  assert.equal(counts['劉慈欣'], 2);
  assert.equal(counts['村田沙耶香'], 1);
});

test('タイトルが空だと作れない', () => {
  assert.throws(() => createBook('', '劉慈欣', 5, '感想'), /タイトルは必須です/);
});

test('評価が範囲外だと作れない', () => {
  assert.throws(() => createBook('三体', '劉慈欣', 0, '感想'));
  assert.throws(() => createBook('三体', '劉慈欣', 6, '感想'));
});

test('保存されていなければ空の一覧になる', () => {
  assert.deepEqual(parseStoredBooks(null), []);
});

test('保存されているJSONを本の一覧に戻せる', () => {
  const json = JSON.stringify([{ title: '三体', author: '劉慈欣', rating: 5, review: '' }]);
  const books = parseStoredBooks(json);
  assert.equal(books.length, 1);
  assert.equal(books[0].title, '三体');
});

test('配列ではないデータが保存されていたら、空の一覧として扱う', () => {
  const json = JSON.stringify({ broken: true });
  assert.deepEqual(parseStoredBooks(json), []);
});
```

<Callout type="warn" title="つまずきポイント: テストファイルの名前と場所">
`node --test`が自動的に拾ってくれるのは、次のどちらかです。

- ディレクトリ名がちょうど`test`(単数形)のフォルダの中にある、すべての`.js`・`.mjs`・`.cjs`ファイル
- それ以外の場所でも、ファイル名が`.test.js`・`.test.mjs`のように**特定の形で終わる**ファイル

hondanaの`tests`フォルダは複数形の名前なので、1つ目のルールには当てはまりません。だから`tests/logic.test.mjs`は、フォルダの中にあることではなく、**ファイル名が`.test.mjs`で終わっていること**によって拾われています。試しに、同じ`tests`フォルダの中に`tests/logic.notes.mjs`のような名前でテストを書いても、`node --test`はそれを見つけてくれません——実行された形跡すら残らない、気づきにくい失敗の仕方です。テストファイルの名前は、必ず`.test.mjs`(または`.test.js`)で終わらせてください。
</Callout>

`createBook`の直し方は、こうです。

```js nocheck
export function createBook(title, author, rating, review) {
  const cleanTitle = (title ?? "").trim();
  const cleanAuthor = (author ?? "").trim();
  if (!cleanTitle) {
    throw new Error("タイトルは必須です");
  }
  if (!cleanAuthor) {
    throw new Error("著者は必須です");
  }
  if (!(rating >= 1 && rating <= 5)) {
    throw new Error(`評価は1から5で入力してください: ${rating}`);
  }
  return { title: cleanTitle, author: cleanAuthor, rating, review };
}
```

## このレッスンで分かったこと

- `node:test`と`node:assert/strict`は、Node.js 22に標準で入っている。追加のインストールは要らない
- `rating < 1 || rating > 5`のような比較は、`NaN`に対して両方`false`になり、範囲外を見逃す。`rating >= 1 && rating <= 5`の形の方が安全
- 検索・集計・検証(`logic.js`)をDOMや`localStorage`から切り離すと、プレーンオブジェクトの配列を渡すだけでテストできる
