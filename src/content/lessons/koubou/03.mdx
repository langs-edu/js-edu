## 冊数だけでは、物足りない

前回までのhondanaは、「2冊、登録されています。」という1行を表示するだけでした。今日のゴールは、1冊ずつタイトル・著者・評価・感想が並ぶ、本物の一覧を画面に作ることです。

## document.createElementで、要素を作る

コース5では、`querySelector`で**既にあるHTML要素**を取ってくる方法を学びました。今日使う`document.createElement`は逆で、**まだどこにも属していない、新しいHTML要素をその場で作ります。**

<HtmlRunner
  client:visible
  storageKey="koubou-03-a"
  height={120}
  initialHtml={`<ul id="list"></ul>
<script>
  const list = document.querySelector('#list');

  const li = document.createElement('li');
  li.textContent = "三体";
  list.appendChild(li);
</script>`}
/>

`document.createElement('li')`で、まだ画面のどこにも表示されていない`<li>`要素を1つ作ります。`textContent`で中身を入れ、最後に`list.appendChild(li)`で、既にある`<ul id="list">`の**子要素として差し込みます。** `appendChild`を呼ぶまでは、この`li`は画面に何も影響しません。「作る」と「画面に足す」が、はっきり2つの手順に分かれているのが`createElement`のやり方です。

## 本の一覧を、ループで組み立てる

1冊分の`<li>`を、`title`・`author`・`rating`をまとめた`<strong>`と、感想の`<p>`の2つで組み立て、それを本の数だけ繰り返します。

<HtmlRunner
  client:visible
  storageKey="koubou-03-b"
  height={160}
  initialHtml={`<ul id="list"></ul>
<script>
  const books = [
    { title: "三体", author: "劉慈欣", rating: 5, review: "スケールがすごい" },
    { title: "お探し物は図書室まで", author: "青山美智子", rating: 4, review: "静かに沁みる" },
  ];

  const list = document.querySelector('#list');

  for (const book of books) {
    const li = document.createElement('li');

    const heading = document.createElement('strong');
    heading.textContent = \`\${book.title} / \${book.author} (評価\${book.rating})\`;

    const review = document.createElement('p');
    review.textContent = book.review;

    li.appendChild(heading);
    li.appendChild(review);
    list.appendChild(li);
  }
</script>`}
/>

1冊ごとに、`<li>`という「容れ物」を作り、その中に`<strong>`と`<p>`を`appendChild`で足し、できあがった`<li>`を`list`に`appendChild`する——この繰り返しです。for-ofは、コース3から使ってきたおなじみの形です。

## 何も登録されていないときの表示

`books`が空の配列だったら、何も表示しないのではなく、「まだ本が登録されていません。」と案内を出します。

<HtmlRunner
  client:visible
  storageKey="koubou-03-c"
  height={100}
  initialHtml={`<ul id="list"></ul>
<script>
  const books = [];
  const list = document.querySelector('#list');

  if (books.length === 0) {
    const empty = document.createElement('li');
    empty.textContent = "まだ本が登録されていません。";
    list.appendChild(empty);
  }
</script>`}
/>

空の一覧をそのまま放っておくと、「読み込みに失敗したのか、まだ0冊なのか」が利用者には区別できません。**空であることも、1つの状態として、はっきり画面に表す**——これはhondanaに限らず、一覧を扱うアプリでいつも大事になる考え方です。

## innerHTMLではなく、textContentを使う理由

ここまでのコードは、すべて`textContent`で中身を入れてきました。`innerHTML`という、よく似たプロパティもありますが、hondanaでは使いません。次の例で、その理由を確かめます。

<HtmlRunner
  client:visible
  storageKey="koubou-03-d"
  height={160}
  initialHtml={`<ul id="list"></ul>
<script>
  const dangerousTitle = "<b>三体</b>";
  const list = document.querySelector('#list');

  const li1 = document.createElement('li');
  li1.textContent = dangerousTitle;
  list.appendChild(li1);

  const li2 = document.createElement('li');
  li2.innerHTML = dangerousTitle;
  list.appendChild(li2);
</script>`}
/>

同じ文字列`"<b>三体</b>"`を入れたのに、`textContent`の方は`<b>三体</b>`という文字がそのまま見え、`innerHTML`の方は太字の「三体」になって表示されます。`innerHTML`は、渡した文字列を**HTMLとして解釈し直す**からです。

<Callout type="warn" title="つまずきポイント: innerHTMLに、利用者の入力をそのまま入れない">
hondanaのタイトルや感想は、フォームから利用者が自由に入力できます。もし`li.innerHTML = book.title`のように書いていたら、誰かが`<img src=x onerror="...">`のような文字列をタイトルとして登録したとき、それがHTMLとして解釈され、**書き込んだ側が意図したJavaScriptが実行されてしまう**可能性があります。これはXSS(クロスサイトスクリプティング)と呼ばれる、代表的なWebの脆弱性の1つです。

`textContent`は、渡した文字列をどこまでも「ただの文字」として扱います。HTMLタグに見える文字列が入っていても、タグとしては解釈されません。**利用者が入力した値を画面に表示するときは、textContentかcreateElementの組み合わせを使う**——これがhondanaの、そしてこの先あなたが書くアプリの基本の約束です。
</Callout>

## hondana.jsを、更新する

これまでの3つのHtmlRunnerの中身を、`hondana.js`にまとめます。`index.html`には、`<p id="count">`のかわりに`<ul id="book-list">`を用意します。

```js title="hondana.js" nocheck
import { loadBooks, saveBooks } from './storage.js';

function createBook(title, author, rating, review) {
  if (!title) {
    throw new Error("タイトルは必須です");
  }
  if (!author) {
    throw new Error("著者は必須です");
  }
  if (rating < 1 || rating > 5) {
    throw new Error(`評価は1から5で入力してください: ${rating}`);
  }
  return { title, author, rating, review };
}

const shelf = [
  createBook("三体", "劉慈欣", 5, "スケールがすごい"),
  createBook("お探し物は図書室まで", "青山美智子", 4, "静かに沁みる"),
];
saveBooks(shelf);

const listEl = document.querySelector('#book-list');

function render(books) {
  listEl.textContent = "";

  if (books.length === 0) {
    const empty = document.createElement('li');
    empty.textContent = "まだ本が登録されていません。";
    listEl.appendChild(empty);
    return;
  }

  for (const book of books) {
    const li = document.createElement('li');

    const heading = document.createElement('strong');
    heading.textContent = `${book.title} / ${book.author} (評価${book.rating})`;

    const review = document.createElement('p');
    review.textContent = book.review;

    li.appendChild(heading);
    li.appendChild(review);
    listEl.appendChild(li);
  }
}

render(loadBooks());
```

```html title="index.html"
<!doctype html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>hondana</title>
</head>
<body>
  <h1>hondana</h1>
  <ul id="book-list"></ul>

  <script type="module" src="hondana.js"></script>
</body>
</html>
```

<Callout type="note" title="このファイルの検証について">
`hondana.js`は`storage.js`と組み合わさってはじめて成立するので、CodeRunnerでは単体実行できません。実際に`hondana`フォルダの中に置いて、Node.js 22で構文を確認したものです。
</Callout>

`render`の先頭にある`listEl.textContent = "";`にも注目してください。空文字を代入すると、それまで`<ul>`の中にあった`<li>`は全部消えます(コース5で学んだ、textContent代入の性質です)。今日はまだ`render`を1回しか呼びませんが、この「まず空にしてから、作り直す」という形が、次のレッスンで生きてきます。

## このレッスンで分かったこと

- `document.createElement(タグ名)`は、まだどこにも属さない新しいHTML要素を作る。`appendChild`で、既にある要素の子として差し込む
- 一覧は、for-ofで1冊ずつ`<li>`を組み立て、`appendChild`していく
- 「本が0冊」も、はっきり画面に表す1つの状態として扱う
- `innerHTML`は文字列をHTMLとして解釈し直すため、利用者の入力をそのまま渡すと危険(XSS)。**textContentかcreateElementの組み合わせを使う**
- `要素.textContent = "";`で、それまでの子要素をすべて取り除ける
