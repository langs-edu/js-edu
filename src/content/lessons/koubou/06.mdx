## 最後の1歩——手渡せる形にする

テストに守られたhondanaは、もう安心して人に見せられます。最後のレッスンでは、プロジェクトとしての体裁を整え、READMEを書き、そしてこの先どこへ歩いていけるかの地図を広げます。

## package.jsonを、1つ置く

コース1で、モジュールを使うファイルには`type="module"`という印が要る、と学びました。Node.js自身にも、よく似た仕組みがあります。プロジェクトの直下に`package.json`という設定ファイルを置き、`"type": "module"`と書いておくと、そのプロジェクトの`.js`ファイルはすべてモジュールとして扱われます。

```json title="package.json"
{
  "name": "hondana",
  "version": "0.1.0",
  "private": true,
  "type": "module",
  "description": "読んだ本を記録する、小さな読書メモWebアプリ",
  "scripts": {
    "test": "node --test"
  }
}
```

`scripts`に`"test": "node --test"`と書いておくと、`node --test`のかわりに`npm test`という短いコマンドでも同じテストを実行できます。`name`・`version`・`description`は、このプロジェクトが何者かを名乗るための、身分証のようなものです。

<Callout type="note" title="誠実な保留: package.jsonがなくても動く理由">
実は、レッスン5までの`node --test`は、`package.json`がなくても動いていました。Node.js 22は、ファイルの中身に`import`や`export`があれば、それだけでモジュールとして扱ってくれる自動判定を持っているからです。`package.json`は必須ではありませんが、「このプロジェクトはどういうものか」を明示しておく、実務でよく使われる形です。
</Callout>

## READMEを書く

READMEは、「はじめてこのプロジェクトを開いた人」に向けた手紙です。**その「はじめての人」には、半年後のあなた自身も含まれます。** 何を作ったか、動かすには何が要るか、どう使うかを、短く書いておきます。

````markdown title="README.md"
# hondana

読んだ本のタイトル・著者・評価・感想を記録する、小さな読書メモWebアプリです。
JavaScript縁日のコース6「小さな道具をつくる」で、1レッスンずつ作りました。

## 必要なもの

- モダンなブラウザ(Chrome・Firefox・Safari・Edgeなど)
- 簡単なローカルサーバー(`type="module"` のスクリプトは `file://` では動かないため)
- テストを実行する場合は Node.js 22 以上

## 動かす

このディレクトリで、ローカルサーバーを立てます。たとえば:

```console
$ npx serve .
```

表示されたURLをブラウザで開くと、hondanaが起動します。

## 使い方

- フォームにタイトル・著者・評価(1〜5)・感想を入力して「登録」を押すと、一覧に追加されます
- 検索欄に文字を打つと、タイトルか著者に一致する本だけに絞り込まれます
- 「削除」ボタンで、その本を一覧から取り除きます
- 記録はブラウザのlocalStorageに保存され、次に開いたときも残っています

## テスト

```console
$ node --test
```

## 構成

- `index.html` — 画面の骨組み
- `hondana.js` — 画面とイベントを結びつける係(`type="module"`)
- `storage.js` — localStorageへの保存・読み込みを担当する係
- `logic.js` — 検索・集計・検証だけを行う、DOMに触れない係
- `tests/logic.test.mjs` — `logic.js` と `storage.js` の、純粋な部分のテスト

## 既知の制限

- 記録はそのブラウザ・そのオリジンだけに保存されます。別のブラウザやシークレットウィンドウでは見えません
- ブラウザの設定でサイトデータを消すと、記録も消えます
- 1つのタブで開いたままにしていると、別のタブで削除した記録が画面に残ることがあります(タブ間の同期はしていません)
````

「何をするものか」「動かすのに何がいるか」「どう動かすか」「知っている制限」の4つがあれば、READMEとしてまず十分です。あとは使いながら、気づいたことを書き足していきます。

## 公開する、という選択肢

hondanaは、`index.html`・`hondana.js`・`storage.js`・`logic.js`という、静的なファイルだけでできています。サーバー側で動くプログラムを、何も必要としません。だから、GitHubにこのフォルダをアップロードし、**GitHub Pages**という無料の機能を有効にするだけで、世界中のどこからでも開けるURLとして公開できます。手順の詳細はGitHubの公式ドキュメントに譲りますが、「サーバーを自分で立てなくても、静的なWebアプリなら手軽に公開できる」ということだけ、ここでは覚えておいてください。

<Callout type="yorimichi" title="npmという場所への入り口">
このレッスンで書いた`package.json`は、実は**npm**という、世界最大級のJavaScriptパッケージの倉庫と繋がる入り口でもあります。今日は`"scripts"`しか使いませんでしたが、npmには他の人が作った便利な道具(パッケージ)を取り込む仕組みもあります。🏮 [屋台のこぼれ話「npmという巨大な倉庫」](../../../yatai/07-npm/)で、その世界の広さと、たった11行のコードが世界中のビルドを止めかけた事件を紹介しています。
</Callout>

## この先の地図

6コースを歩き終えたあなたは、もう「入門書を読む」段階を過ぎています。ここからは、必要なときに必要な場所を訪ねる読み方に変わります。行き先をいくつか置いておきます。

### MDN——最重要のリファレンス

**MDN Web Docs**(developer.mozilla.org)は、JavaScript・HTML・CSS・Web APIについて、ブラウザを作っている当事者たちが中心になって書いている、もっとも信頼できるリファレンスです。`Array.prototype.filter`のような、知っているはずのメソッドでも、迷ったらまずMDNを開く習慣をつけてください。検索エンジンで見つかる誰かのブログ記事より、たいてい正確で、今も更新され続けています。

### 次の一歩: 非同期処理

hondanaは、`localStorage`という、その場ですぐ終わる処理だけでできていました。でも世の中のWebアプリの多くは、サーバーにデータを取りに行く、といった「時間のかかる処理」を扱います。JavaScriptでこれを表す`Promise`と`async`/`await`という仕組みは、今回のコースでは扱いませんでしたが、この先いちばん近い、次の一歩です。名前だけ、ここで覚えておいてください。

### 型を足す、という発想——TypeScript

コース1で、JavaScriptは「型が値のほうについている」動的型付けの言語だと学びました。この性質は自由な反面、`createBook`に数値のつもりで文字列を渡してしまうような間違いを、実行するまで教えてくれません。**TypeScript**は、JavaScriptに型の宣言を足せるようにした言語で、こうした間違いを実行前に検出できます。今では、多くの現場のJavaScriptプロジェクトで標準的に使われるようになっています。🏮 [屋台のこぼれ話「TypeScriptという義兄弟」](../../../yatai/10-typescript/)もあわせてどうぞ。

### 画面を組み立てる道具——フレームワーク

hondanaでは、`document.createElement`や`appendChild`を自分の手で呼んで画面を組み立てました。これは、画面がもっと複雑になったとき、書く分量も、気をつけることも増えていきます。**React**・**Vue**・**Svelte**のようなフレームワークは、「状態が変わったら、画面のどこを描き直すか」を肩代わりしてくれる道具です。今日のhondanaで自分の手で書いた`render`関数は、これらのフレームワークが裏側でやっていることの、いちばん小さな見本でもあります。どれか1つが正解ということはなく、実際に手を動かしながら、自分に合うものを探していくことになります。

### ブラウザを飛び出した、もう1つの世界——Node.js

コース1で少しだけ触れたNode.jsは、ブラウザの外でJavaScriptを動かす場所でした。今回のコースでも、テストを実行するのに`node --test`を使いました。ファイルの読み書きや、サーバーを立てることまで、JavaScriptだけでできてしまう——ブラウザの中にとどまらない、もう1つの広い世界が、この先に待っています。

## 6つのコースを、ふりかえる

コース1で、コンピュータは「察しない、誠実な読み手」だと知りました。`console.log`から始まり、変数、値の種類、テンプレートリテラル、数と計算、そしてエラーの読み方まで——JavaScriptという処理系との、最初の会話でした。

コース2で、その読み手に「判断」を教えました。if、`===`、while、for、そして関数とスコープ。上から下へ、ときどき分かれて、くりかえす——プログラムの骨格です。

コース3で、値をまとめる道具を手に入れました。配列、オブジェクト、分割代入とスプレッド、そして`undefined`と`null`という、2つの「ない」。単語カウンタで、集計というものに初めて触れました。

コース4で、関数が値であるという、JavaScriptの核心に出会いました。アロー関数、コールバック、`map`・`filter`・`find`、class、JSON、そして例外。ミニ図書館で、それらを1つにまとめました。

コース5で、ページという舞台に手を伸ばしました。DOM、`querySelector`、`textContent`、`classList`、そして`addEventListener`。押されたら動く、という体験を初めて作りました。

そしてコース6で、それら全部を使って、hondanaという実物を作りました。設計し、ブラウザに保存し、画面を組み立て、イベントで繋ぎ、テストで固め、手渡せる形にしました。

## このレッスンで分かったこと

- `package.json`の`"type": "module"`は、Node.jsに対して「このファイル群はモジュールだ」と伝える、`type="module"`のNode.js版
- READMEには、何をするものか・動かすのに何が要るか・どう動かすか・知っている制限を書く
- hondanaは静的なファイルだけでできているので、GitHub Pagesのような場所に手軽に公開できる
- この先の地図: MDN(リファレンス)、非同期処理(Promise/async・await)、TypeScript(型)、React・Vue・Svelte(フレームワーク)、Node.js(ブラウザの外の世界)
