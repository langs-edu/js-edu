## 同じアクセスを、何度も書いてきた

前回、`mise.namae`、`mise.nedan`のように、同じ形のアクセスを繰り返し書いてきました。値を1つずつ、名前を指定して取り出す——正しい書き方ですが、キーの数が増えるほど同じような行が並びます。

JavaScriptには、配列やオブジェクトから複数の値をまとめて取り出す、短い書き方があります。**分割代入**です。

## 配列の分割代入

配列の値を、順番どおりに変数へ一気に取り出せます。

<CodeRunner
  client:visible
  storageKey="matomeru-05-a"
  initialCode={`const yatai = ["りんご飴", 300, 12];
const [namae, nedan, zaiko] = yatai;
console.log(namae);
console.log(nedan);
console.log(zaiko);`}
/>

実行すると、下の出力欄にこう表示されます。

```console
りんご飴
300
12
```

`const [namae, nedan, zaiko] = yatai;`は、`yatai[0]`を`namae`に、`yatai[1]`を`nedan`に、`yatai[2]`を`zaiko`に、まとめて代入しています。配列の分割代入は**位置**で対応します。並べる順番を変えると、対応する値も変わります。

## オブジェクトの分割代入

オブジェクトにも同じ発想の書き方があります。こちらは`{ }`を使い、**キーの名前**で対応します。

<CodeRunner
  client:visible
  storageKey="matomeru-05-b"
  initialCode={`const mise = { namae: "たこ焼き", nedan: 500 };
const { namae, nedan } = mise;
console.log(namae);
console.log(nedan);`}
/>

```console
たこ焼き
500
```

配列の分割代入と違い、オブジェクトの分割代入は`{ nedan, namae }`のように**順番を入れ替えても同じ結果**になります。キーの名前がそのまま変数名になるので、対応がずれる心配がありません。

<Callout type="mame" title="別の変数名で受け取りたいとき">
`{ namae: miseNamae }`のように書くと、`namae`キーの値を`miseNamae`という別の名前で受け取れます。すでに`namae`という変数を使っていて名前が衝突するときなどに使います。多用すると読みにくくなるので、必要なときだけの技として覚えておけば十分です。
</Callout>

## for-ofの中で、分割代入する

分割代入は`for-of`とも組み合わせられます。前回の「配列とオブジェクトを組み合わせる」で書いた`mise.namae`・`mise.nedan`は、こう短くできます。

<CodeRunner
  client:visible
  storageKey="matomeru-05-c"
  initialCode={`const yataiList = [
  { namae: "りんご飴", nedan: 300 },
  { namae: "たこ焼き", nedan: 500 },
];
for (const { namae, nedan } of yataiList) {
  console.log(\`\${namae}: \${nedan}円\`);
}`}
/>

```console
りんご飴: 300円
たこ焼き: 500円
```

`for (const { namae, nedan } of yataiList)`は、「取り出した1件のオブジェクトから、`namae`と`nedan`をその場で分割代入する」という意味です。`mise`という名前を経由せず、直接キーの値が使えます。

## スプレッド構文——配列をコピーする

`...`（3つのドット）を使うスプレッド構文は、配列やオブジェクトの中身を**展開**します。よく使うのは、コピーを作りながら値を足す場面です。

<CodeRunner
  client:visible
  storageKey="matomeru-05-d"
  initialCode={`const original = ["りんご飴", "たこ焼き"];
const copy = [...original, "金魚すくい"];
console.log(copy);
console.log(original);`}
/>

```console
[ 'りんご飴', 'たこ焼き', '金魚すくい' ]
[ 'りんご飴', 'たこ焼き' ]
```

`[...original, "金魚すくい"]`は、「`original`の中身を全部展開して、そのあとに`"金魚すくい"`を足した、新しい配列」を作ります。レッスン1の`push`とは違い、**元の`original`は変わりません。** 新しい配列を作りたいときと、元の配列を書き換えたいときとで、使い分けます。

## スプレッド構文——オブジェクトを合成する

オブジェクトにも同じ`...`が使えます。

<CodeRunner
  client:visible
  storageKey="matomeru-05-e"
  initialCode={`const base = { namae: "りんご飴", nedan: 300 };
const updated = { ...base, nedan: 350 };
console.log(updated);
console.log(base);`}
/>

```console
{ namae: 'りんご飴', nedan: 350 }
{ namae: 'りんご飴', nedan: 300 }
```

`{ ...base, nedan: 350 }`は、「`base`の中身を全部展開して、そのあとに`nedan: 350`を上書きした、新しいオブジェクト」を作ります。同じキーが後ろにあると、後ろの値が優先されます。元の`base`はそのまま残り、`updated`という別のオブジェクトができました。

<Callout type="note" title="読めることが大事。多用は控えめに">
分割代入もスプレッド構文も便利ですが、いくつも重ねて使うと、かえって何が起きているか読み取りにくくなります。この縁日では、「繰り返し書いていた同じアクセスを、はっきり短くできる場面」でだけ使い、無理に詰め込まないことをおすすめします。読み手（未来のあなたを含めて）が一目で追える形が、いちばんです。
</Callout>

## このレッスンで分かったこと

- 配列の分割代入`const [a, b] = 配列;`は**位置**で対応する
- オブジェクトの分割代入`const { a, b } = オブジェクト;`は**キーの名前**で対応する。順番は関係ない
- `for (const { a, b } of 配列)`のように、for-ofと組み合わせられる
- スプレッド構文`...`は、配列やオブジェクトの中身を展開してコピーや合成をする。元の値は変わらない
- 分割代入もスプレッド構文も、読みやすさを保てる範囲で使う
