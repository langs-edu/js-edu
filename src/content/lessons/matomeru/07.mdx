## ここまでの道具を、全部使う

前回、`?.`で「ない」を安全に扱えるようになりました。このコースで学んだのは、配列、`for-of`、オブジェクト、ネスト、分割代入、`undefined`と`null`——データをまとめて扱うための道具です。最後に、それを1つの練習にまとめます。

作るのは**単語カウンタ**です。1つの文章を単語に分けて、それぞれの単語が何回登場したかを数えます。

## 文章を、単語に分ける

文字列には`split`というメソッドがあります。指定した文字で文章を区切り、配列にして返します。

<CodeRunner

  initialCode={`const text = "りんご飴 たこ焼き りんご飴 金魚すくい たこ焼き りんご飴";
const words = text.split(" ");
console.log(words);`}
/>

実行すると、下の出力欄にこう表示されます。

```console
[ 'りんご飴', 'たこ焼き', 'りんご飴', '金魚すくい', 'たこ焼き', 'りんご飴' ]
```

`text.split(" ")`は、`text`を半角スペースで区切って、6個の文字列が入った配列にしています。ここから先は、この`words`という配列を相手にします。

## オブジェクトで、数を数える

単語ごとの登場回数を、オブジェクトに集計します。単語そのものをキーに、登場回数を値にします。

<CodeRunner
  client:visible
  storageKey="matomeru-07-b"
  initialCode={`const text = "りんご飴 たこ焼き りんご飴 金魚すくい たこ焼き りんご飴";
const words = text.split(" ");
const counts = {};

for (const word of words) {
  counts[word] = (counts[word] ?? 0) + 1;
}

console.log(counts);`}
/>

```console
{ 'りんご飴': 3, 'たこ焼き': 2, '金魚すくい': 1 }
```

3回登場する`りんご飴`、2回の`たこ焼き`、1回の`金魚すくい`——ちゃんと数えられました。この`counts[word] = (counts[word] ?? 0) + 1;`という1行が、このレッスンの主役です。じっくり見ていきます。

**なぜブラケット記法か。** `word`は`"りんご飴"`や`"たこ焼き"`のように、実行するまで中身が分からない変数です。レッスン3で学んだとおり、キーが変数のときはブラケット記法（`counts[word]`）を使うのでした。

**なぜ`?? 0`か。** `word`が初めて登場したとき、`counts[word]`はまだキーが無いので`undefined`です（レッスン3・6の復習です）。`undefined`のまま`+ 1`しようとすると、期待通りの数値にはなりません。そこで`counts[word] ?? 0`——「`counts[word]`が`undefined`か`null`なら`0`、そうでなければその値」を使うことで、初めての単語は`0 + 1`で`1`から、2回目以降は今までの数に`+ 1`で数えられます。

## ??（Nullish coalescing）という演算子

`??`は**Nullish coalescing**（ヌリッシュ・コアレシング）演算子と呼ばれ、「左側が`undefined`か`null`なら右側を使う、そうでなければ左側をそのまま使う」という意味です。

似た働きをする`||`と比べると、`??`のありがたさが分かります。

<CodeRunner
  client:visible
  storageKey="matomeru-07-c"
  initialCode={`console.log(0 || 5);
console.log(0 ?? 5);`}
/>

```console
5
0
```

コース2で、`0`は**falsy**な値だと学びました。`||`は「左側がfalsyなら右側」なので、`0 || 5`は`0`をfalsyとみなして`5`を返してしまいます。今回のカウンタで`||`を使っていたら、登場回数が`0`回のところで毎回`5`にすり替わり、正しく数えられません。`??`が見ているのは「`undefined`か`null`か」だけです。`0`はどちらでもないので、`0 ?? 5`は`0`のままです。**「まだ無い」と「0という有効な値」を区別したいときは、`||`ではなく`??`を使います。**

## 一番よく出てくる単語を探す

数えながら、同時に「今のところ一番多い単語」も記録できます。レッスン4で書いた「一番高い屋台を探す」と同じ発想です。

<CodeRunner
  client:visible
  storageKey="matomeru-07-d"
  initialCode={`const text = "りんご飴 たこ焼き りんご飴 金魚すくい たこ焼き りんご飴";
const words = text.split(" ");
const counts = {};
let maxWord = "";
let maxCount = 0;

for (const word of words) {
  counts[word] = (counts[word] ?? 0) + 1;
  if (counts[word] > maxCount) {
    maxCount = counts[word];
    maxWord = word;
  }
}

console.log(counts);
console.log(\`最頻出: \${maxWord}（\${maxCount}回）\`);`}
/>

```console
{ 'りんご飴': 3, 'たこ焼き': 2, '金魚すくい': 1 }
最頻出: りんご飴（3回）
```

1つの`for-of`の中で、数えることと、最大を更新することを同時にやっています。単語を数えるたびに、「今数えたばかりの回数が、これまでの最大を超えたか」をその場でチェックしているので、あとから配列やオブジェクトをもう一度たどる必要がありません。

<Callout type="mame" title="単語ごとに数を数える別の道具">
オブジェクトの代わりに`Map`という専用の道具を使う書き方もあります。キーに文字列以外も使えたり、順番が保証されたりと便利な性質がありますが、このコースではまだ扱いません。オブジェクトでの集計は、素朴でも十分実用的な方法です。
</Callout>

## このレッスンで分かったこと

- `文字列.split(区切り文字)`で、文字列を配列に分割できる
- `counts[word] = (counts[word] ?? 0) + 1;`は、オブジェクトを使った集計の定番のかたち
- `??`（Nullish coalescing）は、左側が`undefined`か`null`のときだけ右側を使う。`0`のような有効なfalsy値を巻き込む`||`とは違う
- 1つの`for-of`の中で、集計と同時に最大・最小の追跡もできる
- このコースで学んだ配列・オブジェクト・for-of・分割代入・undefined/nullは、組み合わせて初めて実用的な力になる
