## コース4を、1つにまとめる

コース4では、関数が値であること（レッスン1）、それを短く書くアロー関数（レッスン2）、あとで呼んでもらうコールバック（レッスン3）、配列の道具箱`map`・`filter`・`find`（レッスン4）、同じ形のオブジェクトをまとめる`class`（レッスン5）、データを文字列にして運ぶJSON（レッスン6）、エラーを受け止めたり自分から起こしたりする例外（レッスン7）——と学んできました。

このレッスンでは、それを全部使って、小さな図書館を作ります。本を表す`Book`クラスと、本の集まりを管理する`Library`クラスです。

## Bookクラス——1冊の本を表す

まずは、1冊の本を表す、シンプルなクラスです。

```js
class Book {
  constructor(title, author, rating) {
    this.title = title;
    this.author = author;
    this.rating = rating;
  }

  describe() {
    return `『${this.title}』（${this.author}） 評価:${this.rating}`;
  }
}
```

`title`（タイトル）・`author`（著者）・`rating`（評価、1〜5の整数のつもり）を持ち、`describe()`で読みやすい文字列にします。レッスン5で作った`Yatai`と、形はそっくりです。

## Libraryクラス——本の集まりを管理する

`Library`は、`Book`の配列を持ち、追加・検索・保存・読み込みをまとめて引き受けます。

順番に見ていきます。

- `addBook(book)`——レッスン7と同じ形で、`rating`が1〜5の範囲外なら`throw new Error(...)`します。おかしなデータが図書館に紛れ込むのを、その場で止めます
- `listTitles()`——`this.books`を`map`で「タイトルの配列」に変換します（レッスン4）
- `search(keyword)`——`filter`で、タイトルか著者に`keyword`を含む本だけを残します（レッスン4）
- `findByTitle(title)`——`find`で、タイトルが完全に一致する最初の1冊を探します（レッスン4）
- `toJSON()`——`this.books`を`JSON.stringify`で文字列にします（レッスン6）
- `static fromJSON(json)`——`static`をつけたメソッドは、インスタンスからではなく`Library.fromJSON(...)`のように**クラス自身**から直接呼び出せます。中身は、文字列を`JSON.parse`でいったんただのオブジェクトの配列に戻し、それぞれを`new Book(...)`に**組み直して**から`addBook`する、という処理です

<Callout type="note" title="レッスン6の伏線を、fromJSONで回収する">
レッスン6で、「`JSON.parse`が返すのは、もとのクラスのインスタンスではなく、ただのオブジェクト」だと確認しました。`Library.fromJSON`が`JSON.parse`の結果をそのまま使わず、1件ずつ`new Book(item.title, item.author, item.rating)`で**組み直している**のは、そのためです。ひと手間かけることで、読み込んだデータも、ちゃんと`describe()`が呼べる`Book`のインスタンスに戻ります。
</Callout>

    this.books = [];

## このレッスンで分かったこと

- `Book`と`Library`という2つの`class`で、1冊の本と、本の集まりをそれぞれ表した
- `map`・`filter`・`find`で、タイトル一覧の作成・検索・1件検索をそれぞれ書いた
- `addBook`の中で`throw`し、おかしなデータが入り込む前に止めた
- `JSON.stringify`で保存用の文字列を作り、`static fromJSON`で文字列から`Book`のインスタンスを**組み直して**読み込んだ
- コース4で学んだ7つの道具は、単独でも便利ですが、組み合わせるとさらに大きな仕組みを作れる
