## 関数にも、typeofで型がある

前回までで、配列とオブジェクトを組み合わせてデータをまとめ、`function`で手順に名前をつけて使ってきました。ここまでの関数は、いつも`function 名前(引数) { ... }`という決まった形でした。

コース4では、この関数を少し違う角度から見ます。JavaScriptでは、**関数もまた値の一種**です。数値や文字列と同じように、`typeof`で型を確かめられます。

<CodeRunner
  client:visible
  storageKey="sekkei-01-a"
  initialCode={`function greet(name) {
  return \`こんにちは、\${name}さん\`;
}

console.log(typeof greet);
console.log(greet("ここね"));`}
/>

`typeof greet`は`"function"`を返します。`number`や`string`と並ぶ、れっきとした型のひとつです。「関数は値である」というのが、このレッスンでいちばん伝えたいことです。

<Callout type="mame" title="なぜobjectじゃなくfunctionなの？">
JavaScriptの関数は、実は「呼び出せる性質を持ったオブジェクト」の一種です。それでも`typeof`は`"object"`ではなく`"function"`という専用の答えを返してくれます。呼び出せるかどうかをすぐ見分けられるようにという、使い勝手のための特別あつかいです。
</Callout>

## 関数を、別の名前で呼ぶ

値であるなら、他の値と同じように、別の変数に代入できるはずです。試してみましょう。

<CodeRunner
  client:visible
  storageKey="sekkei-01-b"
  initialCode={`function greet(name) {
  return \`こんにちは、\${name}さん\`;
}

const sayHello = greet;
console.log(sayHello("コウタ"));
console.log(sayHello === greet);`}
/>

`const sayHello = greet;`は、`greet`という関数の値を、`sayHello`という別の名前でも指せるようにしています。`sayHello("コウタ")`は`greet("コウタ")`とまったく同じ意味です。最後の行の`sayHello === greet`が`true`になることからも、2つの名前が同じひとつの関数を指していると分かります。数や文字列に別名をつけるのと、何も変わりません。

## 名前のない関数——関数式

ここまでは、先に`function greet(name) { ... }`という宣言があって、それを別名で呼び直しただけでした。でも、関数の値がほしいだけなら、そもそも名前をつけた宣言を経由する必要はありません。

<CodeRunner
  client:visible
  storageKey="sekkei-01-c"
  initialCode={`const greet2 = function (name) {
  return \`ようこそ、\${name}さん\`;
};

console.log(typeof greet2);
console.log(greet2("あかり"));`}
/>

`function`の直後に名前がありません。この、名前を持たない関数の値を作る書き方を**関数式**と呼びます。`greet2`という名前は、あくまで`const`で値につけた変数名であって、関数そのものの名前ではありません。関数式が作る関数のことを、そのまま**無名関数**（名前のない関数）と呼ぶこともあります。

これまでの`function greet(name) { ... }`という書き方は**関数宣言**と呼びます。「宣言」と「式」、似ていますが中身は少し違います。

<Callout type="warn" title="つまずきポイント：関数宣言と関数式で、使える時点が違う">
関数宣言は、ファイルの読み込み時点で先に登録されるため、定義より前の行から呼び出せます。関数式は`const`や`let`への代入なので、変数と同じルールに従い、代入の行より前では使えません。

```js expect-error
console.log(sayLater("ここね"));

const sayLater = function (name) {
  return `あとで、${name}さん`;
};
```

実行すると、次のようなエラーになります（実際にはこの上にファイル名や行番号を示す行も表示されますが、ここではエラーの種類だけ示します）。

```console
ReferenceError: Cannot access 'sayLater' before initialization
```

「先に使える関数宣言」と「代入した行から使える関数式」——この違いを知っておくと、思わぬ`ReferenceError`に出会ったときにあわてずに済みます。
</Callout>

## 関数を、引数として渡す

値であるなら、他の関数の**引数として渡す**こともできます。ここがコース4でいちばん大事な一歩です。

<CodeRunner
  client:visible
  storageKey="sekkei-01-d"
  initialCode={`function operate(a, b, operation) {
  return operation(a, b);
}

const result = operate(3, 4, function (x, y) {
  return x + y;
});

console.log(result);`}
/>

`operate`は、3つ目の引数として**関数そのもの**を受け取り、`operation(a, b)`という形でそれを呼び出しています。呼び出す側は、`operate(3, 4, ...)`のところで「3と4に対して、何をするか」を関数式としてその場で渡しています。渡す関数式を`function (x, y) { return x - y; }`に変えれば、同じ`operate`が引き算にもなります。**「何をするか」を、値として持ち運べる**——これが、関数が値であることの実用的な意味です。

## 関数を、オブジェクトにしまう

関数が値なら、オブジェクトのプロパティにもできます。コース3で学んだドット記法・ブラケット記法が、そのまま使えます。

<CodeRunner
  client:visible
  storageKey="sekkei-01-e"
  initialCode={`const ops = {
  add: function (a, b) {
    return a + b;
  },
  sub: function (a, b) {
    return a - b;
  },
};

console.log(ops.add(10, 3));
console.log(ops["sub"](10, 3));`}
/>

`ops`というオブジェクトの`add`キーにも`sub`キーにも、関数の値が入っています。`ops.add(10, 3)`は「`ops`の`add`プロパティが指す関数を、`10`と`3`を引数にして呼び出す」という意味です。文字列のキーで呼び出す`ops["sub"](10, 3)`も同じように動きます。この「名前と処理をひとまとめにする」形は、レッスン5の`class`につながっていきます。

## このレッスンで分かったこと

- JavaScriptでは、関数もまた値の一種。`typeof`で確かめると`"function"`が返る
- 関数の値は、`const 名前 = 既存の関数`のように別の変数に代入できる
- `function`の後ろに名前を書かない**関数式**で、名前のない関数の値をその場で作れる
- 関数宣言は定義より前から呼べるが、関数式は代入の行より前では使えない（変数と同じルール）
- 関数の値は、他の関数の**引数として渡せる**。「何をするか」を持ち運べることが、関数が値であることの実用的な意味
- 関数の値は、オブジェクトのプロパティにもできる。`obj.method()`という形は、次のレッスン以降でも繰り返し出てくる
