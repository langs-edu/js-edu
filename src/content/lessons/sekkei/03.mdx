## 呼んでもらうために、渡す

前回までで、関数を他の関数の引数として渡せることを見てきました。ここまでの`operate`は、渡した関数をその場ですぐに呼び出していました。でも、渡した関数がいつ呼ばれるかは、渡した側が自由に決めてよいことです。「今すぐ」でも構いませんし、「くり返しの中で、そのつど」でも、「ずっとあとで」でも構いません。

**あとで呼んでもらうために渡す関数**のことを、**コールバック**と呼びます。「呼び返される関数」という意味です。

## 自分のrepeatを作る

`n`回だけ、渡された関数を呼び出す`repeat`という関数を作ってみます。

<CodeRunner
  client:visible
  storageKey="sekkei-03-a"
  initialCode={`function repeat(n, fn) {
  for (let i = 0; i < n; i++) {
    fn(i);
  }
}

repeat(3, (i) => console.log(\`\${i}回目\`));`}
/>

`repeat`自身は、「`n`回くり返して、そのたびに`fn(i)`を呼ぶ」というくり返しの骨組みしか知りません。「呼ばれたときに何をするか」は、まったく知りません。それを決めるのは、呼び出す側が渡すコールバック——ここでは`i`から`${i}回目`という文字列を作って表示するアロー関数——の役目です。骨組みと中身が分かれているのが、コールバックの便利なところです。

## 配列を歩く、自分のforEachItemを作る

同じ考え方で、配列を歩く関数も自作できます。

<CodeRunner
  client:visible
  storageKey="sekkei-03-b"
  initialCode={`function forEachItem(items, callback) {
  for (const item of items) {
    callback(item);
  }
}

const yatai = ["りんご飴", "金魚すくい", "ラムネ"];
forEachItem(yatai, (item) => console.log(\`\${item}が並んでいます\`));`}
/>

`forEachItem`は、配列`items`をfor-ofで歩きながら、要素ひとつひとつを`callback`に渡して呼び出しています。「どう歩くか」は`forEachItem`が決め、「歩きながら何をするか」はコールバックが決める——`repeat`とまったく同じ形です。この形、次のレッスンでもう一度出会います。

## setTimeoutで、「あとで」を覗く

コールバックは、「今すぐ」だけでなく「あとで」呼ぶためにも使えます。ブラウザにもNode.jsにも、指定した時間が経ってからコールバックを呼び出す`setTimeout(fn, ミリ秒)`という関数が最初から用意されています。

<CodeRunner
  client:visible
  storageKey="sekkei-03-c"
  initialCode={`console.log("最初");

setTimeout(() => {
  console.log("あとで");
}, 0);

console.log("次");`}
/>

実行すると、`最初` → `次` → `あとで`の順に表示されます。`setTimeout`に`0`ミリ秒を指定していても、`あとで`が最後になることに注目してください。`setTimeout`は「今すぐ実行する」のではなく、「今書かれている処理がひと通り終わったあとで、コールバックを呼ぶ」という予約をする関数だからです。

<Callout type="note" title="非同期の深入りはしません">
「今書かれている処理が終わったあとで、コールバックが呼ばれる」という順番の仕組みは、JavaScriptの**非同期処理**という大きなテーマの入り口です。この縁日では、コールバックが「今」ではなく「あとで」呼ばれることがある、という感覚だけをここで持ち帰ってください。仕組みの詳しい話には立ち入りません。
</Callout>

## つまずきポイント：fnとfn()はまったく別のもの

コールバックでいちばん多い間違いは、「関数を渡す」つもりで「関数を呼び出した結果を渡してしまう」ことです。

<Callout type="warn" title="つまずきポイント：呼び出しの丸かっこを、うっかりつけてしまう">
次のコードを見てください。`sayHi`を`repeat`に渡したつもりですが、`sayHi()`と丸かっこをつけてしまっています。

```js expect-error
function repeat(n, fn) {
  for (let i = 0; i < n; i++) {
    fn(i);
  }
}

function sayHi() {
  console.log("こんにちは！");
}

repeat(3, sayHi());
```

```console
こんにちは！
TypeError: fn is not a function
```

`sayHi()`と丸かっこをつけると、その場で`sayHi`が**即座に実行**されます（だから`こんにちは！`が1回だけ表示されます）。そして`repeat`の第2引数に渡るのは、`sayHi`という関数そのものではなく、`sayHi()`の**戻り値**——`sayHi`は何も`return`していないので`undefined`です。`repeat`の中で`fn(i)`を呼ぼうとしても、`fn`は`undefined`なので、`TypeError: fn is not a function`になります。

**「あとで呼んでもらう」つもりなら、丸かっこをつけずに関数の値そのものを渡します。** `repeat(3, sayHi)`が正しい形です。
</Callout>

## このレッスンで分かったこと

- あとで呼んでもらうために渡す関数を**コールバック**と呼ぶ
- 自作の`repeat(n, fn)`や`forEachItem(items, callback)`のように、「くり返しの骨組み」と「そのたびに何をするか」を分けられるのが、コールバックの便利なところ
- `setTimeout(fn, ミリ秒)`は、コールバックを「今すぐ」ではなく「あとで」呼ぶ代表例。詳しい仕組み（非同期処理）はここでは扱わない
- `fn`（関数そのもの）を渡すつもりで`fn()`（呼び出した結果）を渡してしまうのは、コールバックの定番の間違い。丸かっこの有無で意味がまったく変わる
