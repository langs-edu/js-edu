## 同じ形のオブジェクトを、何個も作りたい

コース3でオブジェクトを、レッスン1で「関数もオブジェクトに入れられる」ことを学びました。この2つを組み合わせると、次のようなものが書けます。屋台を2つ、それぞれ`name`・`price`・`stock`（在庫）と、`sell`（売る）という関数を持つオブジェクトとして表してみます。

<CodeRunner
  client:visible
  storageKey="sekkei-05-a-obj"
  initialCode={`const appleCandy = {
  name: "りんご飴",
  price: 300,
  stock: 5,
  sell(count) {
    this.stock -= count;
    return this.price * count;
  },
};

const ramune = {
  name: "ラムネ",
  price: 200,
  stock: 10,
  sell(count) {
    this.stock -= count;
    return this.price * count;
  },
};

console.log(appleCandy.sell(2));
console.log(appleCandy.stock);
console.log(ramune.sell(3));
console.log(ramune.stock);`}
/>

オブジェクトリテラルの中では、`sell: function (count) { ... }`の代わりに`sell(count) { ... }`と、`function`を省いた書き方ができます。これを**メソッド記法**と呼びます。中の`this`は、「そのメソッドを呼び出したオブジェクト自身」を指します。`appleCandy.sell(2)`の中の`this`は`appleCandy`、`ramune.sell(3)`の中の`this`は`ramune`です。

動きはします。でも、屋台が3つ、10個と増えていくと、`name`・`price`・`stock`・`sell`をまるごとコピーして貼り付けることになります。`sell`の中身にバグが見つかったら、コピーした数だけ直して回らなければなりません。

## classで、かたちを1つにまとめる

「同じ形のオブジェクトを、いくつも作る」ための専用の書き方が**class**です。

<CodeRunner
  client:visible
  storageKey="sekkei-05-b-class"
  initialCode={`class Yatai {
  constructor(name, price, stock) {
    this.name = name;
    this.price = price;
    this.stock = stock;
  }

  sell(count) {
    this.stock -= count;
    return this.price * count;
  }

  describe() {
    return \`\${this.name}（\${this.price}円、残り\${this.stock}個）\`;
  }
}

const appleCandy = new Yatai("りんご飴", 300, 5);
const ramune = new Yatai("ラムネ", 200, 10);

console.log(appleCandy.sell(2));
console.log(appleCandy.describe());
console.log(ramune.sell(3));
console.log(ramune.describe());`}
/>

1つずつ見ていきます。

- `class Yatai { ... }`——`Yatai`という名前の**かたち**を定義します。クラス名は`CapWords`（先頭を大文字にする書き方）にします
- `constructor(name, price, stock) { ... }`——`new Yatai(...)`で新しいオブジェクトを作るときに、最初に1回だけ呼ばれる特別なメソッドです。受け取った引数を`this.name = name;`のように、そのオブジェクト自身のプロパティに詰めていきます
- `sell(count) { ... }`と`describe() { ... }`——ふつうのメソッドです。書き方は、さっきのオブジェクトリテラルの中の`sell(count) { ... }`と同じ**メソッド記法**です
- `new Yatai("りんご飴", 300, 5)`——`Yatai`というかたちに沿って、実際のオブジェクト（**インスタンス**と呼びます）を1つ作ります。`new`を忘れずに

`appleCandy`と`ramune`は、同じ`Yatai`というかたちから作られていますが、それぞれ別の`name`・`price`・`stock`を持つ、別々のオブジェクトです。`appleCandy.sell(2)`は`appleCandy`の`stock`だけを減らし、`ramune`には影響しません。

<Callout type="warn" title="つまずきポイント：newを忘れると呼び出せない">
`class`から作ったコンストラクタは、`new`をつけずに関数として呼び出そうとするとエラーになります。

```js expect-error
class Yatai {
  constructor(name, price, stock) {
    this.name = name;
    this.price = price;
    this.stock = stock;
  }
}

const broken = Yatai("たこ焼き", 400, 3);
console.log(broken);
```

```console
TypeError: Class constructor Yatai cannot be invoked without 'new'
```

`function`宣言と違い、`class`のコンストラクタは`new`をつけて呼ぶこと以外を許しません。「`new`をつけ忘れたら、その場でちゃんと教えてくれる」——これはJavaScriptにしては珍しく、親切な部類のエラーです。
</Callout>

## thisの安全な書き方

`this`は奥が深く、書き方によっては「呼び出したオブジェクト自身」ではなく別のものを指してしまうことがあります。特に、メソッドをアロー関数（`sell: (count) => { ... }`のような形）で書くと、期待した`this`にならない場合があります。

<Callout type="note" title="このサイトの安全な流儀">
この縁日では、`class`の中のメソッドは、すべて`名前(引数) { ... }`という**メソッド記法**で書きます。アロー関数をメソッドとして使うことはしません。この形を守っていれば、`this`は「そのメソッドを呼び出したオブジェクト自身」に安定して結びつきます。`this`の細かい仕組みには、ここでは立ち入りません。
</Callout>

<Callout type="mame" title="classは、あとから入った見た目">
JavaScriptに`class`という書き方が入ったのは、比較的最近のことです。それより前から、JavaScriptには「同じ形のオブジェクトを作る」ためのしくみ（**プロトタイプ**と呼ばれます）がありました。`class`は、その裏の仕組みの上に乗った、読みやすい見た目（構文糖と呼ばれます）です。プロトタイプそのものの説明は、このレッスンでは扱いません。名前だけ、頭の片隅に置いておいてください。
</Callout>

## このレッスンで分かったこと

- `class 名前 { ... }`で、同じ形のオブジェクトを作るための**かたち**を定義できる
- `constructor(引数) { ... }`は、`new`で作るときに最初に1回だけ呼ばれ、`this.プロパティ = 値`で初期値を詰める
- `new クラス名(引数)`で、そのかたちに沿った**インスタンス**を作る。`new`を忘れるとTypeErrorになる
- メソッドは`名前(引数) { ... }`という**メソッド記法**で書く。このサイトでは、`this`を安定させるため、メソッドをアロー関数では書かない
- `class`は比較的あとから入った見た目で、裏の仕組み（プロトタイプ）は別にある。詳細はここでは扱わない
