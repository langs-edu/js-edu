## functionと書くのが、少し長く感じてきたら

前回、`operate(a, b, operation)`に関数式をその場で渡しました。

```js nocheck
const result = operate(3, 4, function (x, y) {
  return x + y;
});
```

渡したいのは「`x`と`y`を受け取って、`x + y`を返す」というただ1行の処理なのに、`function`・引数の丸かっこ・波かっこ・`return`と、まわりの飾りが多く感じます。この「その場で渡す小さな関数」を短く書くための専用の書き方が、**アロー関数**です。

## =>で書く

同じ処理をアロー関数で書き直すと、こうなります。

<CodeRunner
  client:visible
  storageKey="sekkei-02-a"
  initialCode={`function operate(a, b, operation) {
  return operation(a, b);
}

const result = operate(3, 4, (x, y) => {
  return x + y;
});

console.log(result);`}
/>

`function (x, y) { ... }`が`(x, y) => { ... }`になりました。`function`という単語が消え、引数リストの直後に`=>`（矢印）が入っただけです。中身の書き方——波かっこと`return`——は、そのままです。

## 式が1つだけなら、波かっことreturnを省ける

アロー関数の本領は、ここからです。関数の中身が「1つの式を計算して、その結果を返すだけ」なら、波かっこと`return`を省略できます。

<CodeRunner
  client:visible
  storageKey="sekkei-02-b"
  initialCode={`function operate(a, b, operation) {
  return operation(a, b);
}

const result = operate(3, 4, (x, y) => x + y);

console.log(result);`}
/>

`(x, y) => x + y`——これだけで「`x`と`y`を受け取り、`x + y`を返す関数」になります。波かっこがないときは、`=>`の右側に書いた式の値が、そのまま関数の戻り値になります。`return`を書く必要すらありません。

<Callout type="warn" title="つまずきポイント：波かっこをつけたらreturnが必要">
波かっこ`{ }`をつけた時点で、それは「複数の文を書ける本体」になります。中で何かを計算しても、`return`を書かなければ戻り値は`undefined`です。波かっこなしの省略形にだけ、自動で返す特典がついています。
</Callout>

## 引数が1つなら、丸かっこも省ける

引数がちょうど1つのときは、その引数を囲む丸かっこも省略できます。

<CodeRunner
  client:visible
  storageKey="sekkei-02-c"
  initialCode={`const shout = (word) => word + "！";
const shout2 = word => word + "！";

console.log(shout("こんばんは"));
console.log(shout2("こんばんは"));
console.log(shout("こんばんは") === shout2("こんばんは"));`}
/>

`(word) => ...`も`word => ...`も、まったく同じ意味です。このサイトでは、読みやすさのために丸かっこをつける書き方を基本にしますが、省略した形もよく見かけるので、読めるようにしておいてください。**引数が0個、または2個以上のときは、丸かっこを省略できません。**

<CodeRunner
  client:visible
  storageKey="sekkei-02-d"
  initialCode={`const now = () => "縁日は夜からです";
console.log(now());`}
/>

引数が0個のときは、`()`という空の丸かっこが必要です。省いてよいのは「丸かっこの中身」ではなく、「引数がちょうど1つのときの、丸かっこそのもの」だけです。

## つまずきポイント：オブジェクトを返したいとき

省略形には、有名な落とし穴が1つあります。オブジェクトをそのまま返したいときです。

<Callout type="warn" title="つまずきポイント：{ }がオブジェクトなのか本体なのか、区別がつかない">
次のコードは、一見オブジェクトを返しているように見えますが、実際には`undefined`が返ります。

```js
const makeStallWrong = (name, price) => { name, price };

console.log(makeStallWrong("りんご飴", 300));
```

```console
undefined
```

`=>`の直後の`{`は、「本体の始まり」と解釈されてしまいます。中の`name, price`は、オブジェクトのつもりで書いても、`,`で区切られた2つの式をただ評価するだけの文になり、`return`もないので何も返りません。**エラーにはならず、静かに`undefined`が流れます。**JSらしい、止まって教えてくれない例のひとつです。

正しくオブジェクトを返すには、全体をもう一段、丸かっこで囲みます。

```js
const makeStall = (name, price) => ({ name, price });

console.log(makeStall("りんご飴", 300));
```

```console
{ name: 'りんご飴', price: 300 }
```

`({ ... })`と丸かっこで囲むことで、「これは本体のブロックではなく、丸かっこの中の式（オブジェクトリテラル）です」とJavaScriptに伝えられます。オブジェクトを返す省略形のアロー関数を書くときは、この丸かっこを忘れないでください。
</Callout>

## 使い分けの流儀——functionとアロー、どちらを使うか

ここまでで、関数式もアロー関数も、どちらも関数の値を作れると分かりました。この縁日では、次の流儀で書き分けます。

- **トップレベルの独立した手順**（プログラムの主な処理のまとまり）は、これまで通り`function`宣言で書く
- **他の関数にその場で渡す、小さな関数**は、アロー関数で書く

`operate`のような「独立して意味を持つ手順」は`function`のままにして、`(x, y) => x + y`のような「渡すためだけの小さな処理」をアロー関数にする、という使い分けです。これは文法上の決まりではなく、読みやすさのための、このサイトの流儀です。

## このレッスンで分かったこと

- アロー関数は`(引数) => 本体`という形で関数式を短く書く書き方
- 本体が式1つだけなら、波かっこと`return`を省略できる。式の値がそのまま戻り値になる
- 引数がちょうど1つのときは、丸かっこも省略できる。0個や2個以上のときは省略できない
- 波かっこの省略形でオブジェクトを返すときは、`(name, price) => ({ ... })`のように、オブジェクト全体を丸かっこで囲む。囲まないと`{ }`が本体のブロックと解釈され、静かに`undefined`が返る
- このサイトでは、トップレベルの独立した手順は`function`宣言、他の関数に渡す小さな関数はアロー関数、という流儀で書く
